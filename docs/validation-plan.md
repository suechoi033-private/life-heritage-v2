## 1. 목표 분해 — 20명을 A/B/C/D로 어떻게 배분하나

### 1-1. 판단 기준에서 역산한 "완료 인터뷰" 목표

판단 기준(`interview-guide.md`):
- A(부모 돌봄) 5명 중 3명+ → 입구 가설 확정
- B(사별) 5명 중 3명+ → lifecycle 전환 가설 확정
- C(비혼·1인) 구체 트리거+행동이면 가설 확정

→ **판단 기준을 충족하려면 A 최소 5명, B 최소 5명, C 최소 3명의 "완료 인터뷰"가 필요**하다.

| 분류 | 완료 인터뷰 목표 | 무엇을 함 | 비고 |
|---|---|---|---|
| A. 페르소나1 (부모 돌봄) | **5명** | 인터뷰(Part1) + 사용성(Part3) | 판단기준 5명 분모 충족 필수 |
| B. 페르소나2 (사별) | **5명** | 인터뷰(Part2) + 사용성(Part3) | 판단기준 5명 분모 충족 필수 |
| C. 페르소나3 (비혼·1인) | **3~4명** | 인터뷰(Part4) + 사용성(Part3) | 트리거+행동 신호 확인 |
| D. 비페르소나 | **6~7명** | **사용성(Part3)만** | 앱 사용성 검증 표본 + 페르소나 분류 오판 흡수 |
| **합계** | **20명** | | |

### 1-2. 모집 목표 역산 (no-show·노쇼·분류 오판 반영)

완료 20명을 확보하려면 더 많이 섭외해야 한다. **수락률 60%, no-show/이탈 20%** 가정.

```
완료 20명 ÷ (수락률 0.6 × 출석률 0.8) ≈ 42명 섭외 발송 필요
보수적으로 ▶ "45명 섭외 발송" 을 모집 목표로 잡는다.
```

| 단계 | 목표 수 | 환산 |
|---|---|---|
| 후보 명단 확보 | 50명 | 풀에서 추림 |
| 섭외 메시지 발송 | **45명** | 명단의 90% |
| 일정 확정 (수락) | 27명 | 발송×60% |
| 완료 인터뷰/테스트 | **20명** | 확정×80%(노쇼 흡수) |

> 핵심: A·B는 페르소나 조건이 까다로워 풀에 적다. **A 후보·B 후보를 먼저 확실히 5명씩 확보**하는 게 병목. C·D는 상대적으로 모으기 쉽다.

## 3. 섭외 운영

### 3-1. 섭외 메시지 템플릿 (복붙용 — 발송은 창업자 승인 후)

**① 인터뷰용 (A·B·C 후보에게)**
```
[이름]아/님, 잘 지내지?
나 요즘 '죽음·노후·돌봄을 미리 준비하도록 돕는 서비스'를 만들고 있어.
네 경험을 30분만 들려주면 정말 큰 도움이 될 것 같아서 연락했어.
정답 없는 편한 대화고, 평가 아니야. 그냥 네가 겪은 얘기 듣는 거야.
이번 주말이나 다음 주 중에 30분, 통화/화상/만나서 중 편한 걸로 가능할까?
부담되면 거절해도 전혀 괜찮아 🙂
```

**② 사용성 테스트용 (D 후보에게)**
```
[이름]아/님, 부탁 하나만!
내가 만들고 있는 웹서비스가 있는데, 처음 보는 사람 눈에 어떤지 30분만 봐줄 수 있어?
화면 보면서 "이게 뭐 같아? 여기서 뭐 할 수 있을 것 같아?" 정도 솔직하게 말해주면 돼.
잘 만들었다는 칭찬 말고, 헷갈리는 데를 찾는 게 목적이라 막 헤매줄수록 좋아.
이번 주말~다음 주 중 30분 가능할까?
```

### 3-2. 부담 줄이는 법 (수락률·진실성 동시 확보)
- "30분", "편한 채널", "거절해도 OK" 3종 세트를 항상 넣는다.
- 보상은 소액 기프티콘(커피) 정도 고지 가능 — 단 **지출은 창업자 승인**, 그리고 보상은 "솔직한 칭찬"을 유도하므로 과하지 않게.
- 인터뷰 목적을 "내 서비스 평가"가 아니라 "네 경험 듣기"로 프레이밍 → 아첨 신호 줄임(질문지 원칙과 일치).

### 3-2b. 사용성 테스트 표준 동선 (D 전원 + 인터뷰 대상 공통, Part3)

> 말로 설명하지 말고 직접 만지게 둔다. 막히는 지점이 곧 개선점. 실제 메뉴명 기준.

| 순서 | 시키는 것 (실제 메뉴) | 관찰 포인트 |
|---|---|---|
| 1 | 홈 첫 화면만 5초 보여주고 "이거 무슨 서비스 같아요?" | 첫인상·이해 여부 |
| 2 | "둘러봐 주세요" 손 안 대고 관찰 | 어디를 누르나, 어디서 멈추나 |
| 3 | **케어링(nest/care)** → "면회 일지 한 번 남겨보세요" | 끝까지 가나, 어디서 막히나 |
| 4 | **커뮤니티(forest)** → 글 1개 열어 "끝까지 읽고 싶어요? 왜?" | 콘텐츠 흡인력 |
| 5 | (앱 정리 전이면 발자취·유언·셀프는 "공사중, 패스") | 오염 신호 차단 |
| 6 | "다시 들어올 것 같아요? 어떤 상황이면?" / "주변에 필요할 사람?" | 재방문·추천 = 진짜 가치 신호 |

### 3-3. no-show 대비
- 확정 후 **하루 전 리마인드** 1회(시간·링크 재전송).
- 슬롯을 목표보다 20% 더 잡는다(확정 27 → 완료 20 가정).
- 노쇼 시 즉시 같은 분류의 대기 후보에게 재섭외(명단에 "대기" 표시).
