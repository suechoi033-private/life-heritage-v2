## 0. 이 문서가 존재하는 이유

오늘의집은 한국 최고 수준의 제품 실행 조직이다. 그들의 자료에서 배울 것은 **일하는 규율**이지 **전환 최적화 전술**이 아니다.

| | 오늘의집 | 잇다 |
|---|---|---|
| 목적 | GMV·전환율(CVR) | 후회를 줄인다 |
| 동력 | 욕망·긴급·경쟁 | 신뢰·동행·존엄 |
| 모수 | 수천만, 얕게 | 수천, 30년 깊게 |
| 사용자 | 쇼핑 중, 빠르게 훑음 | 케어링·사별 중, 시니어, 천천히 |

→ **같은 메커니즘이라도 정신이 반대면 우리에겐 독이다.** 카운트다운 타이머는 오늘의집에선 전환 도구지만, 잇다에선 사별자에게 압박을 거는 잔인함이다. 이 문서의 전부는 그 분리를 지키기 위해 있다.

### 북극성: "60점의 시대"에 우리가 던질 질문

> 버킷플레이스 디자인 블로그 *"누구나 60점을 만드는 시대에 프로덕트 디자이너가 던져야 할 진짜 질문"*(2026-06-12)에서 자극받음. (원문 접근 차단 상태라 제목 + 같은 조직의 1차 증거로 재구성함 — 원문 확보 시 갱신.)

AI가 누구나 **60점(평균치 UI·카피·플로우)**을 즉시 찍어주는 시대다. 같은 조직의 디자이너가 직접 적었듯 — "디자인 퀄리티가 혁신적이진 않습니다. **결국 보는 안목이 필요해요**"(김라인, Claude Design 후기). 

→ **차별점은 더 이상 '잘 만드는 손'이 아니라 '무엇을·왜 만드는가를 던지는 질문과 판단(안목)'이다.** 이것이 이 문서가 *전술(60점은 AI가 준다)*보다 *메서드와 판단 원칙(60점을 90점으로 올리는 안목)*에 무게를 둔 이유다. 잇다에서 그 "진짜 질문"은 항상 하나로 수렴한다:

> **"이게 사람들이 덜 후회하게 하는가?"** (헌장 가치2)

AI에게 화면을 시키기는 쉽다. 사별자에게 무엇을 보여주지 *않을지*, 어떤 침묵을 지킬지, 언제 다음 단계로 손을 내밀지를 판단하는 것 — 그게 60점을 잇다로 만드는 안목이고, AI가 대신 못 하는 우리의 일이다.

## 1. 메서드: 오늘의집에서 그대로 훔칠 7가지

이것들은 톤과 무관한 **순수 실행 규율**이다. 잇다도 똑같이 한다.

### M1. 모든 변경은 `문제 → 가설 → 솔루션`으로 적는다
오늘의집의 모든 진행공유가 이 골격이다. 솔루션부터 그리지 않는다. "무슨 문제를, 어떤 사용자에게, 어떤 증거로 풀고, 무엇이 관측되면 성공인가"를 먼저 쓴다. 헌장의 "가설은 증거로 검증한다"와 동일. → 잇다의 모든 기능 PR/문서는 이 헤더를 단다.

### M2. 만들기 전에 4~9명에게 보여준다 (퀵 UT)
그들은 "오메이커스 UT 6인", "오유저데이 4인"으로 **출시 전** 시안을 깐다. 이것이 그들의 속도의 비밀이다. 잇다는 더 그래야 한다(시니어·민감 주제). 헌장 "5명 인터뷰"의 운영판. → 새 플로우는 지인 3~5명 손에 쥐여보기 전에 배포 금지.

### M3. 정보 위계는 강박적으로 다듬는다
"height 최소화", "정보 밀도", "뱃지 덜어내기" — 화면의 모든 요소가 자기 자리를 증명해야 한다. **단, 잇다는 방향이 반대다**: 그들은 더 많이 욱여넣어 탐색량을 늘리고, 우리는 **덜어내 한 가지에 집중**시킨다(시니어 가독성). 규율은 같고 벡터만 반대.

### M4. ATF(첫 화면)가 전부다
"광고로 랜딩한 25%가 ATF만 보고 이탈." 첫 스크롤 없이 보이는 영역에서 "여기가 어디고 나는 무엇을 하면 되는가"가 끝나야 한다. → 잇다 모든 페이지: 스크롤 전에 단계·다음 행동이 보이는가?

### M5. 데드엔드를 만들지 않는다 (다음 행동 항상 제시)
"주문완료 페이지가 Dead End", "검색 결과 없음 시 이탈" → 그들은 모든 끝에 다음 경로를 깐다. **잇다에선 이게 곧 lifecycle 엔진이다**(아래 P4). 한 화면의 끝 = 막다른 길이 아니라 다음 단계로의 다리.

### M6. 입력 비용을 줄인다 (prefill·skip·단계 노출)
해외통관 플로우: "바텀시트 대신 같은 영역 하단 확장", "키보드 자동 호출", "프리필 분기". 건자재 담당자 지정: "최대한 많은 케이스에서 직접입력 안 해도 넘어가게." → 잇다의 사전연명·일지·초대처럼 **귀찮으면 안 하는** 고마찰 과업에 그대로 적용.

### M7. 패턴을 늘리지 않는다 (기존 것 재사용)
"장기보관상품 UI 동일하게 사용 — 신규 UI 대비 구현 비용 낮고 변수 최소화." 새 컴포넌트를 만들기 전에 있는 패턴으로 푼다. → 잇다 디자인시스템 v3 토큰·반경·카드 형태 안에서 해결. 인라인 hex·일회용 컴포넌트 금지(design-system.md 규칙과 동일).

## 2. 전술: 버리거나 뒤집을 것

오늘의집의 전환 전술 대부분은 잇다 헌장 직접 위반이다. 표로 못박는다.

| 오늘의집 전술 | 잇다 처분 | 이유 / 뒤집은 버전 |
|---|---|---|
| 카운트다운 타이머("오늘 23:59 마감") | **금지** | 사별·케어링 중인 사람에게 시간 압박은 잔인. 긴급은 우리 동력이 아니다 |
| "n명이 보고있어요" / "최근 n명 구매" | **금지** | 경쟁·과시 = 헌장이 의도적으로 걷어낸 것. 좋아요 수 경쟁을 없앤 것과 같은 이유 |
| 소셜프루프 랭킹·인기순 경쟁 | **뒤집기** | 경쟁이 아니라 **동행**으로: "비슷한 시기를 지나는 중 N명"(전략 E카드). 숫자로 줄세우지 않고 "혼자가 아니다"만 전한다 |
| 업셀·크로스셀("이것도 사세요") | **뒤집기** | 매출 유도가 아니라 **lifecycle 다음 단계 안내**로(P4). "장례 후 90일 안에 할 7가지" = 같은 메커니즘, 반대 정신(후회 감소 봉사) |
| 뱃지 남발(특별인증가·오늘출발·Pick…) | **최소화** | 신뢰 라벨은 딱 2종만(전략 확정): `✓ 잇다 에디터` / `먼저 겪은 분의 기록` |
| "최대 혜택" 같은 과장 워딩 | **금지** | 신뢰는 천천히 쌓고 한 번에 무너진다(헌장 가치4). YMYL은 더더욱 |
| 다크패턴(받기 유도·바텀시트 강제) | **금지** | 그들조차 "다크패턴 감시 기간"이라 적음. 잇다는 처음부터 담백하게 |
| 무한 피드(구경 최적화) | **반려** | 이미 결정됨(전략 6-B): 시니어가 길 잃음 → **허브+피드 하이브리드**. 순수 무한 피드 X |

**한 줄 원칙: 메커니즘이 "더 사게/더 머물게"를 위한 것이면 의심하고, "덜 후회하게/덜 외롭게"로 번역되지 않으면 버린다.**

## 3. 잇다 프로덕트 디자인 원칙 7 (P1~P7)

위 메서드·전술 분리를 잇다 고유 원칙으로 응축한 것. **모든 화면 결정의 기준선.**

### P1. 조용함이 기본값이다 (Quiet by default)
움직이는 것·빛나는 것·재촉하는 것을 먼저 의심한다. 모션·배지·강조색은 "없으면 길을 잃는가?"를 통과해야 산다. 에버그린(`--primary`)은 주요 행동 하나에만. 클레이(`--warm`)는 케어링의 미세 악센트로만. 화면 대부분은 화이트·명조·여백.

### P2. 시니어가 첫 사용자다 (Senior-first, not senior-friendly)
40~60대가 부모 케어링 중에, 사별 직후 흐린 정신으로 쓴다. → 한 열·큰 글씨(`body zoom 1.1` 유지)·**제목 + 한 줄 요약**·터치 영역 넓게·전문용어 0. "친절하게도"가 아니라 **기본 설계 대상**이 시니어다. 정보는 욱여넣는 게 아니라 덜어낸다(M3 반대 벡터).

### P3. 한 화면, 한 가지 (One screen, one thing)
ATF에서 "지금 내 단계 + 다음 한 가지 행동"이 끝나야 한다(M4). 선택지를 늘어놓지 않는다. 사별자에게 12개 버튼은 폭력이다. 메뉴는 글 하나하나가 진입로(전략 확정).

### P4. 끝은 다음 단계로의 다리다 (Lifecycle bridge, not dead-end)
**잇다에서 가장 중요한 원칙.** 모든 화면의 끝에는 다음 lifecycle 단계로의 자연스러운 안내가 있다(M5를 미션으로 승격). 단, 업셀이 아니라 **transition trigger nudge**:

- 일지 100개 쌓임 → "이 기록, 책으로 남기시겠어요?"
- 사별 등록 → "장례 후 90일 안에 해야 할 7가지"
- 사별 후 6~12개월 → "그때 가장 막막했던 거… 내 가족엔 안 그러게"

이게 30년 lifecycle SaaS를 굴리는 엔진이다(전략 3절 Transition Trigger). 오늘의집의 크로스셀 기술 = 잇다의 후회 감소 봉사.

### P5. 관계가 리텐션이다 (Relationship is the retention)
잇다의 유일한 구조적 해자 = "형제·가족이 같은 페이지를 본다"(전략 5절 ③). → **초대·공유·공동 작성**은 부가기능이 아니라 1급 시민. 입력 비용을 0에 가깝게(M6), 데드엔드 없이(M5), 로그인으로 막지 않고(P6). 이 한 플로우가 사업의 사활이다.

### P6. 로그인은 관문이 아니라 맥락 보존 장치다 (잭의 규칙)
오늘의집 잭의 통찰을 그대로 채택: 로그인을 요구하는 기준은 아래 중 **2개 이상일 때만**.
1. 되돌릴 수 없는 행동인가? (결제·최종 등록)
2. 결과가 사용자 자산인가? (내 기록·사전연명 문서)
3. 로그인 없이는 물리적으로 불가능한가?

→ 콘텐츠 읽기·둘러보기는 로그인 없이. 탐색 중간에 가입을 들이밀어 사별자를 쫓아내지 않는다. SEO 유입(전략 1순위)을 가입벽으로 죽이지 않는다.

### P7. 카피의 톤이 제품의 톤이다 (Microcopy carries the dignity)
오늘의집조차 "마감(부정) → 지금 주문시 출발(긍정)"으로 한 단어를 바꿔 전환을 올렸다. 잇다는 단어 하나에 존엄이 걸린다. → 긴급·과장·경쟁 워딩 금지. 민감 문구는 카피라이터(`life-heritage-copywriter`) 마감 필수. "위로 말고, 같이"(전략 톤).

## 4. 화면 개편 인사이트 (실제 페이지 매핑)

위 원칙을 현재 잇다 화면에 적용했을 때 **바꾸면 좋을 것들**. 우선순위(★ = 사업 영향 큼) 표시.

### ★★★ I1. 형제 초대 플로우 (`invite.html`) — 사업의 사활
리텐션 해자(P5)인데 가장 약할 위험. 오늘의집 메서드 총동원:
- **입력 비용 0화(M6):** 카톡 공유 한 번 = 초대 완료. 가입 강요 없이 "보기 링크" 먼저, 참여는 나중에(P6).
- **데드엔드 금지(M5):** 초대 보낸 뒤 빈 화면 금지 → "형제가 들어오면 여기서 함께 일지를 봐요" 상태 안내.
- **민감 처리(P7):** "초대"라는 단어가 부담이면 "형제에게 이 기록 보여주기"로. 죽음·돌봄 맥락에서 초대는 민감 — 카피라이터 마감.
- 이 플로우는 **무조건 퀵 UT(M2)**. 전략 10절 "검증 안 된 가정 1·형제 초대"의 핵심 검증 지점.

### ★★★ I2. Lifecycle transition nudge 시스템 (전 화면 공통) — 미구현
P4의 실체. 현재 홈은 최신순 피드뿐, 단계 전환 안내가 없다. → 사용자 lifecycle 단계(전략 C 데이터모델) 기반으로 "다음 한 가지"를 각 화면 끝에 1개만 노출. **오늘의집 주문완료 크로스셀 UI 패턴을 그대로 빌리되, 상품 대신 다음 단계 콘텐츠/도구.** 단계 데이터모델(전략 C) 선행 필요.

### ★★ I3. 홈 피드 (`index.html`) — 위계·신뢰 라벨
- **제목 + 한 줄 요약 중심(P2)**, 사진 그리드 X(이미 전략 확정). 첫 카드 hero.
- **신뢰 라벨 2종 통일(전술표):** `✓ 잇다 에디터` / `먼저 겪은 분의 기록`. 그 외 뱃지 금지.
- **경쟁 신호 제거 점검:** 공감 수는 "비슷한 시기를 지나는 중 N명"(동행)으로만. 좋아요 랭킹·인기순 정렬 금지(전술표).
- 추모 카테고리 홈 제외 유지(보류 상태).

### ★★ I4. 사전연명·체크리스트 (`plan-write.html`, `self.html`) — 고마찰 YMYL
끝까지 완주가 KPI인데(전략 10절 가정4) 중도이탈 위험 최고.
- **단계적 노출 + 진척 표시(M6):** 오늘의집 해외통관 "단일 영역 연속 처리" 패턴 — 한 화면에서 한 단계씩, 진행 상태(요청/입력/완료)를 같은 맥락에 노출. 페이지 점프 최소화.
- **skip + 나중에(M6):** 모르면 건너뛰고 근사치 허용, 나중에 수정.
- **YMYL 신뢰 장치(헌장 가치4):** 의사·법무사 監修 명단 visible. 終活ねっと 실패 회피(전략 8절).

### ★★ I5. 오늘의 질문 (`questions.html`, `ask.html`) — 동행 UX
오늘의집 CMS 피드백루프(이미지 퀴즈 + 선택률 즉시 노출)의 **정신을 뒤집어** 채택:
- 답변 후 "비슷한 답을 한 분들이 있어요" 정도의 **동행 신호만**. 선택률을 "남들과 얼마나 겹치나"식 경쟁/정답 프레임으로 쓰지 않는다(전술표).
- 매일 1개·축적 → 회고록 자동 정리로 연결(P4 다리). 4단계 페르소나 엔진.

### ★ I6. 케어 일지 (`care.html`, `care-dashboard.html`) — 빈 상태·공유
- **첫 진입 빈 상태(M5):** "아직 기록이 없어요"로 끝내지 말고 "오늘 면회는 어땠나요?" 한 줄 입력으로 바로 시작.
- **형제 공유 1급 시민(P5):** 일지마다 공유가 자연스럽게(데드엔드 없이). 이게 리텐션.
- 정보 위계: 최근 변경(약·소견·결정)이 ATF에(M4).

### ★ I7. 온보딩 (`onboarding.html`, `welcome.html`) — 로그인 타이밍
- **P6 적용:** 둘러보기 먼저, 가입은 "내 자산(기록·사전연명)이 생기는 순간"에만. 입구에서 가입 강요 금지.
- 진입 동기 후크는 ①②(획득), 리텐션 약속은 ③(함께 쓰기)로 분리해 카피(전략 5절).

## 5. 체크리스트 (PR/디자인 리뷰 시 통과 기준)

새 화면·기능을 내기 전 자문:

- [ ] **문제→가설→솔루션**으로 적혔는가? (M1)
- [ ] 지인 3~5명에게 보여줬는가? (M2)
- [ ] ATF에서 "지금 단계 + 다음 한 가지"가 끝나는가? (M4/P3)
- [ ] 이 화면의 끝이 데드엔드가 아니라 다음 단계 다리인가? (M5/P4)
- [ ] 시니어가 한 열·큰 글씨·한 줄 요약으로 읽는가? (P2)
- [ ] 긴급·경쟁·과장·뱃지남발이 없는가? (전술표/P1)
- [ ] 관계(형제·가족 공유)를 막거나 누락하지 않았는가? (P5)
- [ ] 로그인을 불필요하게 요구하지 않는가? (P6)
- [ ] 토큰만 썼는가? 인라인 hex·일회용 컴포넌트 없는가? (M7/design-system)
- [ ] 민감 카피가 카피라이터 마감을 거쳤는가? (P7)
- [ ] (앱 동작 변경 시) `sw.js` `CACHE_VERSION` 올렸는가?
