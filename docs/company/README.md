# 라이프헤리티지 팀 — 사용법 (Quick Start)

> 새 세션에서 이 팀을 소집하는 법. 창업자(또는 오케스트레이터 세션)가 읽는다.

## 팀 멤버

| 호출 이름 | 역할 | 언제 부르나 |
|---|---|---|
| `life-heritage-strategy` | 사업전략 | 방향·우선순위·BM·"이거 해야 하나?" |
| `life-heritage-ops` | 운영 | 일정·KPI·실행계획·"언제까지 뭘로 측정?" |
| `life-heritage-marketing` | 마케팅 | 콘텐츠·채널·카피·PR |
| `life-heritage-pe` | Product Engineer | 화면·기능·배포 |

## 부르는 법
- 새 세션에서 과제를 주면, 메인(오케스트레이터)이 적절한 에이전트를 소집한다.
- 명시적으로: "전략가에게 물어봐", "PE한테 이 화면 고쳐달라고 해" 식으로 지정 가능.
- 여러 명을 병렬로: "전략·마케팅 둘 다 의견 줘" → 동시 소집 후 교차 비평.

## 표준 워크플로 (큰 과제)
1. **전략**이 방향·성공기준·가설을 잡는다.
2. **마케팅 / PE / 운영**이 각자 실행안.
3. 서로 **교차 비평** 1회.
4. **운영**이 일정·KPI로 묶고 **창업자 승인 필요 항목**을 표시.
5. 결정·결과를 `docs/`에 기록.

## 황금 규칙 (창업자 승인 필수)
돈 · 외부 발송(게시·이메일·DM) · 계약 · 채용 · 법인/세무 · 프로덕션 배포 · 되돌리기 어려운 모든 것.
→ 팀은 여기까지 **초안·계획·코드**를 만들고 멈춘다. 실행은 창업자가.

## 지금 우선순위 (수시 갱신)
1. **지인 20명 검증** (`docs/interview-guide.md`) — 페르소나·지불의향·사용성
2. **시드 콘텐츠 작성** (`docs/strategy.md`의 시드 10개) — 마케팅 주도
3. 검증 결과로 추모(memorial) 방향 확정
