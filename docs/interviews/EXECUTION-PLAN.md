## 1. 목표 분해 — 20명 A/B/C/D 배분

판단 기준(`interview-guide.md`)에서 역산:

| 분류 | 페르소나 | 완료 목표 | 시키는 것 | 사업가설 매핑 |
|---|---|---|---|---|
| A | 부모 케어링 중 (40·50대) | **5명** | 인터뷰 Part1 + 사용성 Part3 | a 형제 지불의향 / b 진입 / c 도구 완주 |
| B | 사별 경험 (50대+) | **5명** | 인터뷰 Part2 + 사용성 Part3 | b 케어링→자기준비 전환 / c |
| C | 비혼·1인(20·40대 여성) | **3~4명** | 인터뷰 Part4 + 사용성 Part3 | (활성시장 정성신호) |
| D | 비페르소나 — **30대 중심 + 20대 일부** | **6~7명** | 사용성 Part3만 | c (활성시장 정성신호) |
| **합** | | **20명** | | |

- 가설 d(매칭 파트너)는 별도 트랙. 본 라운드는 a·b·c.
- (결정 ④) **20·30·40·50대 각 2~3명** 분포 확보로 활성 가설 정성 신호. 30대는 D에서 의식적으로 잡음 (A·B·C에 자연 슬롯 없음). 40·50대는 A·B에 충분, 20대는 C에 일부.

**모집 funnel 역산** (수락 60%, 출석 80%):
```
후보 명단 50 → 발송 45 → 일정 확정 27 → 완료 20
```

**최우선 병목**: A·B 각 5명 후보 확보. 명단 작성 시 A·B 칸부터 채운다 — 못 채우면 검증 전체가 막힌다.

## 3. 섭외 운영

### 3-1. 섭외 메시지 (창업자 검토 후 복붙용)

**① 인터뷰용 (A·B·C 후보)**
```
[이름]아/님, 잘 지내?

나 요즘 '죽음·노후·돌봄을 미리 준비하도록 돕는 서비스'를 만들고 있어.
네 경험을 30분만 들려주면 정말 큰 도움이 될 것 같아서 연락했어.

정답 없는 편한 대화고, 평가 아니야. 그냥 네가 겪은 얘기 듣는 거야.
이번 주말이나 다음 주 중에 30분, 통화/화상/만나서 중 편한 걸로 가능할까?
부담되면 거절해도 전혀 괜찮아 :)
```

**② 사용성 테스트용 (D 후보)**
```
[이름]아/님, 부탁 하나만!

내가 만들고 있는 웹서비스가 있는데, 처음 보는 사람 눈에 어떤지
30분만 봐줄 수 있어?

화면 보면서 "이게 뭐 같아? 여기서 뭐 할 수 있을 것 같아?"
정도 솔직하게 말해주면 돼.
잘 만들었다는 칭찬 말고, 헷갈리는 데를 찾는 게 목적이라
막 헤매줄수록 좋아.

이번 주말~다음 주 중 30분 가능할까?
```

**③ (결정 ④) 20~50대 활성 콘텐츠 반응용 — 30대 중심**
```
[이름]아, 짧게 하나만!
'후회 없이 사는 법'에 대한 글/질문을 만들고 있는데,
본인 또래가 보면 어떤 느낌인지 30분만 솔직하게 말해줄 수 있어?
정답 없어, 그냥 네 반응이 궁금해.
```

### 3-2. 부담 줄이기 (수락률·진실성 동시 확보)

- "30분 / 편한 채널 / 거절해도 OK" 3종 세트 항상 포함
- 보상: **소액 기프티콘(커피, ~5,000원)** — 결정 ② 참조
- 프레이밍: "내 서비스 평가"가 아니라 **"네 경험 듣기"** → 아첨 신호 감소

### 3-3. 사용성 테스트 표준 동선 (Part 3 — 전원 공통)

> 말로 설명하지 말고 **직접 만지게** 둔다. 막히는 지점이 곧 개선점.
> **검증 가능한 3축으로 한정**(결정 ③ 권장안 기준). 미완 영역은 "공사중 — 패스" 안내.

| # | 화면 | 시키는 것 | 보는 것 |
|---|---|---|---|
| 1 | 홈 5초 | "이게 무슨 서비스 같아요?" | 5초 첫인상 / 이해 여부 |
| 2 | 홈 | "둘러봐 주세요" (손 대지 마라) | 어디를 누르나 / 어디서 멈추나 |
| 3 | **케어링(nest)** | 부모 돌봄 기록(면회·약 등) 1건 남기기 | 끝까지 가나 / 어디서 막히나 |
| 4 | **콘텐츠 1편** | 페르소나 맞는 글 한 편 끝까지 읽기 (A→90일/B→사별→자기준비/C→사전연명·디지털) | 정독 / 다 읽었나 / 왜 멈췄나 |
| 5 | **자기성찰 질문**(홈 "오늘 잇고") | 답 쓰고 저장까지 | 저장 도달 / 글쓰기 부담 |
| 6 | 마무리 | "다시 들어올 것 같아요? 어떤 상황이면?" / "주변에 필요할 사람?" | 재방문·추천 = 진짜 가치 신호 |

미완·우려 영역(발자취·유언·셀프·장례희망 등)은 "공사중 패스" — 검증 신호 오염 방지.

### 3-4. no-show 대비
- 확정 후 **하루 전 리마인드 1회** (시간·링크 재전송)
- 슬롯을 목표보다 20% 더 잡기 (확정 27 → 완료 20)
- 노쇼 시 즉시 같은 분류 대기 후보 투입

### 권장 이유 (재검토용 보존)

- **①**: MVP 검증은 창업자가 직접 신호를 받는 게 가장 정확. 부하 분산은 W3 진입 시 재검토.
- **②**: 평일 시작이 창업자 페이스 안정. 보상은 "30분 시간값"의 가벼운 사회적 부채 정도가 진실성·수락률 둘 다 살림.
- **③**: 미완 영역 노출은 신호 오염. 3축이 가장 안정적이고 페르소나별 차별 신호도 잘 나옴.
- **④**: 본 라운드 신호를 흩뜨리지 않으면서 활성 가설 정성 신호도 같이 본다.
- **⑤**: 지인 명단은 sensitive PII라 공개 레포에 못 올림. 시트에서 PII 관리, 주간 요약·결정만 레포로 가져와 단일 원본 유지.
