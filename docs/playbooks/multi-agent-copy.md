# 플레이북 — 멀티에이전트 카피 선별

제목·헤드라인·CTA·퀴즈 문항·푸시 문구처럼 **한 줄이 첫인상이자 전환을 가르는 카피**를
정할 때 쓰는 절차. 한 사람(또는 한 에이전트)의 머리로 쓰면 톤이 편향되고, 죽음·사별
주제에선 그 편향이 곧 사람을 튕겨낸다. 그래서 **여러 관점이 격리된 채 제안 → 한 명이
기준으로 선별**하는 구조를 쓴다.

> 출처: 오늘의집 데이터팀의 Continuous Research System(R1 제안 → R2 EM 선별) 패턴을
> 잇다의 카피 작업에 맞게 가져온 것. ML 학습·메트릭 부분은 버리고 "다양성 확보 →
> 기준 선별"이라는 뼈대만 취했다.

## 언제 쓰나
- 에세이 제목/도입부, 랜딩 헤드라인, CTA 버튼, 가입 퀴즈 문항, 빈 화면·에러 문구, 푸시 알림.
- "그냥 하나 써줘"가 아니라 **고를 가치가 있는** 한 줄일 때. 마이크로카피 한 줄 고치는 건 그냥 카피라이터 직접.

## 언제 안 쓰나
- 본문 장문 에세이 집필(이건 카피라이터가 끝까지 씀), 단순 오타·어휘 통일, 이미 톤이 확정된 반복 문구.

---

## 절차

### R1 — 격리 제안 (관점별로 따로)
각 관점이 **서로의 안을 보지 않고** 독립적으로 N안(기본 3~5안)을 낸다. 집단사고·수렴 방지.

| 관점 | 누가 | 무엇을 노리나 |
|---|---|---|
| **카피라이터** | `life-heritage-copywriter` | 단어·리듬·호흡. "위로 말고, 같이"의 정공법 톤. |
| **마케팅** | `life-heritage-marketing` | 검색·공유·전환. 채널에서 클릭되는 각, SEO 키워드 결. |
| **(선택) 역발상** | 카피라이터에게 별도 지시 | 일부러 다른 결 — 더 조용하게 / 더 구체적인 장면 / 질문형 등. explore용. |

- 입력은 **무엇을 위한 카피인지 + 맥락(화면·독자·바로 앞뒤 문구)** 만. 다른 관점의 결과물은 입력하지 않는다.
- 각 안에는 **노린 감정·톤·길이감**을 한 줄로 붙인다(나중에 고를 근거).

### R2 — 선별 (한 명이 기준으로)
나온 안 전체(보통 8~15안)를 **하나의 기준표**로 채점해 최종 2~3안을 고르고, 그중 추천 1안을 근거와 함께 정한다. 선별자는 보통 카피라이터(톤이 최우선이므로) 또는 창업자.

채점 4축 (각 1~5점):

| 축 | 질문 |
|---|---|
| **톤 적합성** | 잇다 목소리("위로 말고 같이", 조용함·존엄)에 맞나? 신파·훈계·종교색·공포 없나? |
| **명료·전환** | 시니어가 한 호흡에 읽히나? 다음 행동이 분명한가? |
| **차별성** | 클리셰("소중한 순간")가 아니라 장면이 보이나? 잇다만의 결인가? |
| **리스크(역점수)** | 죽음 어휘를 너무 일찍/직격으로 쓰나? 감수 없는 YMYL 단정인가? 첫인상에서 튕길 위험은? |

선별 규칙:
- **exploit 1~2안 + explore 1안.** 안전하게 맞는 안과, 모험적이지만 배울 게 있는 안을 섞어 남긴다(테스트 후보).
- 한 관점이 최종을 독식하지 않게 — 동점이면 다른 관점 안을 올린다.
- **버린 안 중 "이래서 못 쓴다"가 분명한 건 [버린 것 원장](../decisions/REJECTED.md)에 한 줄.** 다음에 같은 결을 또 제안하지 않게.

### 산출물 형식
```
## <대상> 카피 — YYYY-MM-DD

### 후보 (관점 / 안 / 노린 톤)
- [카피] A. "<문구>" — 정공법, 8자, 차분
- [카피] B. "<문구>" — 장면형, 첫 화면 직격 피함
- [마케팅] C. "<문구>" — 검색 의도형, 키워드 'OOO'
- [역발상] D. "<문구>" — 질문형, 더 조용하게
...

### 점수표 (톤/명료/차별/리스크↓)
| 안 | 톤 | 명료 | 차별 | 리스크 | 합 | 비고 |
...

### 최종
- 추천(채택): B — <왜 이게 최선인지 한 줄>
- 테스트 후보(explore): D
- 버림: C(검색엔 좋으나 톤이 광고체 → REJECTED 기록)
```

## 한 번에 돌리는 법 (창업자용)
한 줄로 요청하면 된다. 예:

> "seed-11 살아서 하는 장례식, **제목 후보 멀티에이전트로** 뽑아줘.
>  카피·마케팅 각 4안, 역발상 1안, 선별까지."

오케스트레이터가 카피라이터·마케팅 에이전트를 **병렬로** 띄워(서로 격리) 제안을 모으고,
선별표까지 만들어 추천 1안 + 테스트 후보 1안을 돌려준다.
확정 전 카피는 **창업자 검토 대상**, 외부 발행은 창업자 승인 영역(기존 규칙 그대로).

## 주의 (잇다 고유)
- **죽음 어휘는 첫인상에서 직격 금지.** 삶의 언어로 열고 무거운 단어는 신뢰가 쌓인 뒤. (카피라이터 원칙과 동일)
- **공포·죄책감·가짜 긴급 금지.** "준비 안 하면 가족이 고통받는다", 카운트다운 류는 리스크 축에서 자동 탈락.
- **YMYL 단정은 감수 전 발행 금지.** 카피가 의료·법률·재정을 단정하면 선별에서 배제.
- 좋아요·경쟁 수를 부추기는 카피는 헌장 위반 — 후보에서 제외.
