> 원칙 근거: `product-design-principles.md` M2(만들기 전 4~9명)·P5(관계=리텐션)·P6(보기 먼저)·M5(데드엔드 금지)·P7(존엄 카피)

## 0. 이 키트가 푸는 단 하나의 병목

해자③(관계=리텐션)은 잇다의 **유일한 구조적 해자**인데 **가장 검증이 안 됐다.** 전략은 위험을 이렇게 못박았다:

> 진짜 장벽이 UI가 아니라 **"형제에게 기록을 들이미는 관계적 마찰"**일 수 있다.
> 형제 초대가 민감·귀찮아 거의 안 일어나면, 잇다는 결국 "혼자 쓰는 도구"로 남고 해자는 무너진다.

그래서 이 UT의 설계 핵심은 **두 갈래를 분리해서 판정**하는 것이다:

- **A. UI 문제** — 보내는 법을 몰라서/버튼을 못 찾아서/카톡 메시지가 안 눌려서 → 화면을 고치면 풀린다.
- **B. 관계 마찰** — 방법은 알지만 **형제에게 보내기가 부담스러워서** 안 보낸다 → 화면을 고쳐도 안 풀린다. 제품 방향(누구에게/언제/무엇을 공유로 제안하나)을 다시 본다.

**미발송자에게서 A인지 B인지를 반드시 정성으로 캐는 것**이 이 키트의 존재 이유다. "안 보냄"을 그냥 실패로 적고 넘기면 이 UT는 무의미하다.

## 1. 목적 · 검증 질문 (이 UT로 무엇을 yes/no 판정하나)

> 이 UT는 **소규모 대면/원격 관찰(3~5명)**이다. 정량 본지표(발송률 30%, 7일 내 공동행동)는 전략의 20명 행동 추적(`retention-test-report.sql`)에서 나온다. 이 키트는 그 정량 게이트를 **"왜"로 설명**하고, 발송 전 화면의 막힘·거부감을 먼저 잡는 선행 장치다.

| # | 검증 질문 | 무엇이 관측되면 yes |
|---|---|---|
| VQ1 | **발신자**가 일지를 쓴 뒤, 막힘 없이 가족 초대 발송까지 도달하는가? (UI 통과) | 5명 중 4명+가 힌트 없이 발송 버튼까지 도달, 평균 멈칫 1회 이하 |
| VQ2 | 발신자가 **실제로 보낼 마음이 드는가**, 아니면 도달은 했는데 "근데 안 보낼 것 같아요"라 하는가? (관계 마찰) | "지금 이걸 진짜 형제에게 보낼 것 같다"가 5명 중 3명+ |
| VQ3 | **수신자**가 카톡 링크를 받은 형제 입장에서 **눌러볼 것 같은가**? (가장 중요) | 카톡 미리보기 문구를 보고 "이거 오면 눌러볼 것 같다"가 5명 중 3명+ |

> **VQ2·VQ3가 이 UT의 진짜 무게중심이다.** VQ1(UI)은 통과해도, VQ2/VQ3(마찰)에서 막히면 화면을 아무리 고쳐도 해자는 안 선다. 셋을 분리해 적어야 다음 액션이 갈린다(6절 분기표).

## 2. 대상 · 인원 · 세팅

### 누구를 (3~5명)
- **형제 공동 케어링자**: 부모님이 요양원·요양병원·재가 케어 중이고, **형제·자매가 1명 이상** 있어, 케어 정보를 나눠야 하는 40~60대.
- `interview-guide.md`의 **A 분류(페르소나 1)**와 동일 풀. 인터뷰를 이미 한 사람이면, 이 UT를 그 끝에 10~15분 붙여도 된다(중복 인원 효율).
- **제외**: 형제가 아예 없는 외동(공동 케어링 맥락 자체가 없음 — 이 가설 검증 대상 아님). 단 사별 직후 흐린 상태인 분은 무리해 부르지 않는다.

### 인원이 3~5명인 이유
M2(만들기 전 4~9명)의 최소판. 발송 전 화면의 멈칫·거부감은 3~5명에서 거의 다 드러난다. **정량 30%/7일 공동행동 판정은 여기서 하지 않는다** — 그건 20명 행동 추적의 몫. 여기서 사람 수로 결론 내리지 말 것.

### 진행 방식
- **대면 권장**(시니어·민감 주제 — 표정·손이 멈추는 지점이 데이터). 어려우면 화상 + 화면 공유.
- 진행자는 **옆에서 관찰만**. 손대지 않고, 설명하지 않고, 막혀도 5초는 기다린다. "원래 어떻게 하실 것 같아요?"만 묻는다.
- **유도신문 금지**: "여기 누르면 되죠?", "편하시죠?" ❌. "지금 무슨 생각 하셨어요?", "이 화면 보고 뭘 하면 될 것 같아요?" ⭕.

### 준비물 (창업자 사전 세팅)
- [ ] 테스트용 잇다 계정(발신자 역할용) — 일지가 0~1개만 있는 상태(쓰는 과정을 보기 위해).
- [ ] 발신자 단말(참가자가 직접 만질 폰 또는 공유 화면).
- [ ] **수신자 역할용 카톡 화면** — 발신자가 만든 링크가 실제 카톡 메시지(미리보기 카드 포함)로 어떻게 보이는지 보여줄 화면. 미리 한 번 자기 폰에 보내 캡처/실물 준비.
- [ ] 카톡 공유가 막힐 경우 대비 **"링크로 보내기"** 동선도 점검(코드상 카카오 미설정 시 링크 버튼만 노출됨 — 그 상태도 테스트 가능).
- [ ] 관찰 기록지(4절 체크리스트 인쇄 또는 표).
- [ ] 동의: "화면 조작을 지켜보고 메모해도 될까요. 평가가 아니라 화면을 고치려는 거예요." (녹화는 별도 동의)

### 소요 시간
1인 **20~30분** (일지 작성 5분 + 발송 태스크 5분 + 수신자 태스크 5분 + 질문 10분).

## 3. 진행 시나리오 (태스크)

> 진행자는 각 태스크 시작 시 **목표만 한 줄로 주고**, 방법은 말하지 않는다. 막히는 지점이 곧 개선점이다.

### 사전 (1분) — 맥락 세팅
"부모님 케어를 형제분과 나눠 하고 계신 상황이라고 생각해 주세요. 방금 면회를 다녀온 참이에요. 제가 옆에서 보기만 할게요. 막히셔도 괜찮아요 — 그게 저희가 고쳐야 할 부분이에요."

### 태스크 1 — 발신자: 일지 쓰기 (관찰 5분)
- 지시: **"오늘 면회 다녀온 걸 한 줄 남겨 보세요."**
- 관찰: 빈 상태에서 어디로 손이 가나. 입력을 시작하는 데 망설임이 있나(무엇을 적어야 할지). — 일지 자체 UT가 아니므로 깊게 안 판다. 발송 진입점에 자연스럽게 닿는지가 관심.

### 태스크 2 — 발신자: 가족 초대 발송 (관찰 5분 · VQ1·VQ2)
- 지시: **"이 기록을 형제분과 같이 보고 싶다면, 어떻게 하실 것 같아요?"** (※ "초대 버튼 누르세요" 금지 — 스스로 찾는지를 본다)
- 발송 화면(`invite.html` 발급 모드)에 도달하면:
  - 관찰: "가족과 같이 봐요" 문구를 어떻게 받아들이나. **개인 메시지 칸(한마디)**에 손이 가나, 그냥 넘기나. 카톡/링크 중 무엇을 고르나.
  - 발송 직전 멈추면 한 번만 묻는다(중립): **"지금 무슨 생각 드셨어요?"**
- **결정적 정성 질문(태스크 끝에 반드시):**
  - "방금 이 링크를, 지금 실제로 형제분께 보내실 것 같으세요, 아니면 좀 망설여지세요?"
  - 망설이면: **"보내는 방법이 헷갈려서예요, 아니면 보내는 게 좀 부담스러워서예요?"** ← **A(UI) vs B(관계) 가르는 핵심 분기 질문.**

### 태스크 3 — 수신자: 카톡 받은 형제처럼 열어보기 (관찰 5분 · VQ3 — 가장 중요)
- 전환: "이번엔 반대로, 형제분이 이 카톡을 **받은 입장**이라고 생각해 주세요."
- 수신자 역할용 카톡 화면(미리보기 카드 + 문구)을 보여주고:
  - 지시: **"이런 메시지가 카톡으로 왔어요. 보시고 평소처럼 행동해 주세요."** (누르라고 시키지 않는다 — 누르나 안 누르나가 데이터)
  - 관찰: 미리보기만 보고 **무엇이라 이해하나**("이게 뭐 같으세요?"). 누르나, 망설이나, 넘기나.
  - 링크를 열면(미가입 수신 화면): **보낸 사람 이름 + 한마디 + "들어가기/로그인"**이 보임. 가입을 강요받는 느낌인지, "일단 보여주는구나" 안심이 드는지 관찰(P6 검증).
- **결정적 정성 질문:**
  - "이 카톡이 진짜 형(누나/동생)한테서 왔다면, 눌러보실 것 같으세요? 솔직하게요 — 안 누를 것 같다도 좋은 답이에요."
  - 안 누른다면: **"문구가 뭔가 걸려서예요, 아니면 이런 걸 형제끼리 주고받는 게 어색해서예요?"** ← 다시 A vs B 분기.

## 4. 관찰 포인트 체크리스트 (진행자가 보면서 √)

> 참가자가 말하지 않아도 **몸이 말하는 것**을 적는다. 멈칫 = 멈춘 위치 + 추정 이유 메모.

**발신자 — 발송 도달(VQ1)**
- [ ] 일지 후 "함께 보기/초대" 진입점을 **스스로** 찾았나 / 못 찾고 헤맸나 (어디서?)
- [ ] "이 기록, 가족과 같이 봐요" 카드의 뜻을 **첫눈에** 이해했나
- [ ] 카톡 vs 링크 — 무엇을 골랐나, 망설였나
- [ ] **개인 메시지(한마디) 칸을 썼나** / 비웠나 / 못 봤나 (썼다면 뭐라고? — 직접 인용)
- [ ] 발송 후 상태 안내("보낸 링크, 잘 가 있어요")를 봤나 — 데드엔드 느낌은 없었나(M5)

**발신자 — 보낼 마음(VQ2 · 관계 마찰)**
- [ ] "초대 / 가족 / 함께 보기" 워딩에 **거부감·머뭇거림** 신호가 있었나 (표정·말)
- [ ] 망설임의 정체 = **A(방법 모름)** / **B(보내기 부담)** — 둘 중 어디로 답했나 (직접 인용)
- [ ] "형제에게 내 기록을 보여주는 것" 자체에 대한 코멘트가 나왔나 (관계·서운함·통제 등)

**수신자 — 눌러볼까(VQ3)**
- [ ] 카톡 미리보기만 보고 **무엇이라 이해**했나 (광고/스팸 의심? 가족 일? 모르겠음?)
- [ ] 눌렀나 / 망설였나 / 안 눌렀나 (이유 직접 인용)
- [ ] 미가입 수신 화면 — **"보기 먼저" 가 안심을 줬나**, 아니면 "가입부터 하라네" 거부감이 들었나 (P6)
- [ ] "들어가기 vs 로그인" 두 버튼 앞에서 길을 잃었나

## 5. 질문지

> 5점 척도(1 전혀 아니다 ~ 5 매우 그렇다) + 개방형. **칭찬("좋네요")은 신호 아님 — 흘려듣고, 구체적 행동·이유만 적는다.** `interview-guide.md`와 톤 통일(과거·구체 행동 우선, 유도 금지).

### 5-A. 발신자 공통 (전원)
1. 방금 이 화면으로 형제에게 기록을 같이 보자고 보내는 게, 얼마나 쉬웠나요? **(1~5)** — 어디가 걸렸어요?
2. "초대"·"가족과 같이 봐요"라는 말, 어떤 느낌이었어요? 부담스럽거나 어색한 데 있었어요? (개방형)
3. (한마디 칸을 봤다면) 직접 한마디 적는 칸이 있던데, 적고 싶으셨어요/아니었어요? 적는다면 뭐라고 적으실 것 같아요?
4. 지금 평소 형제와 부모님 정보를 어떻게 주고받으세요? (카톡 단톡? 전화? 안 함?) — 이 화면이 그걸 대신할 것 같아요, 아니면 따로일 것 같아요? *(현재 행동 = 진짜 신호)*

### 5-B. **보낸 사람 / 보낼 사람**용 — 발송 의향이 선 경우
5. 방금 만든 링크, 오늘 안에 실제로 형제분께 보내실 것 같아요? **(1~5)** — 누구한테 제일 먼저요?
6. 보낸다면, 형제분이 그걸 받고 어떻게 반응할 것 같아요? (열어볼까/무시할까/물어볼까)
7. 이 기능 때문에 잇다를 다시 열어볼 것 같은 상황이 있다면 언제예요? *(LQ1과 연결 — 행동 진술)*

### 5-C. **안 보낸 / 안 누른 사람**용 — 관계 마찰 캐기 (★ 이 UT의 핵심)
> 여기서 절대 설득하거나 변호하지 않는다. "안 보낼 것 같다"는 **가장 값진 답**이다. 진행자는 끄덕이며 캔다.

8. 방금 "지금 실제로 보내진 않을 것 같다"고 하셨는데 — 좀 더 들려주실 수 있어요? 무엇이 망설여지게 했어요? (개방형, 충분히 침묵 허용)
9. 그게 **(a) 보내는 방법이 헷갈려서**예요, **(b) 형제에게 이런 걸 보내는 게 부담/어색해서**예요, 아니면 **(c) 둘 다/다른 이유**예요? — 골라주시고, 이유를 말씀해 주세요. **← A/B 판정의 결정 문항**
10. (b를 골랐다면) 어떤 부담이에요? 예를 들어 — "내 방식대로 한다고 잔소리 들을까봐" / "형제가 관심 없을까봐" / "굳이 앱까지 끌어들이는 게 유난스러워 보일까봐" / 그 외? (보기는 예시일 뿐, 본인 말로)
11. 그럼 어떤 상황이거나 어떻게 바뀌면 "이건 보내도 되겠다" 싶으실 것 같아요? (있으면 — 관계 마찰을 푸는 단서)
12. (수신자 역할에서 안 눌렀다면) 형제한테 온 거라도 안 누를 것 같다고 하셨는데, 무엇이 가장 걸렸어요? 문구예요, 아니면 이런 걸 받는다는 것 자체예요?

## 7. 진행자 금기 (재확인)
- 위로하지 않는다. 설득하지 않는다. "이거 좋죠?" 묻지 않는다.
- 막혀도 5초 기다린다. 손대지 않는다.
- "안 보낼 것 같다 / 안 누를 것 같다"는 **실패가 아니라 가장 값진 데이터** — 반갑게 받고 끝까지 캔다.
- 죽음·돌봄 맥락에서 참가자가 감정적으로 흔들리면 **즉시 멈추고** 태스크보다 사람을 우선한다.
