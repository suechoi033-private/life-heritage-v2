> 목적: 현재 흐름을 한 판에 정리하고, 재전략 논의의 기준점으로 삼는다.
> 와이어프레임(시각본): `/ux-review.html` (배포 링크로 열람)

## 1. 현재 동작 (코드 기준 사실)

상단바는 `nav.js`가 그린다. 우측 액션은 두 개:

| 아이콘 | title | 실제 행선지 | 이어지는 화면 |
|---|---|---|---|
| 📚 | "안내" | `forest.html?category=death_prep` | 커뮤니티 페이지 → **콘텐츠 세그먼트 → "죽음 준비" 카테고리 필터** 피드. 카드 탭 → `content-detail.html`. 세그먼트를 커뮤니티로 바꾸면 `post-detail.html`. |
| ✏️ | "오늘 일기" | `seed.html?tab=diary&action=new` | seed.html 로드 → **즉시 `diary-write.html`로 리다이렉트**(작성 화면). |

- 📚는 `infoIcon`이 기본 true라 로그인/비로그인 모두에 노출.
- ✏️는 로그인 + quickWrite일 때만.

## 2. 발견된 문제 (UX)

### 📚 "안내"
1. **아이콘–라벨–행선지 3중 불일치 (High).** 책 아이콘은 보통 "라이브러리/저장/읽을거리/도움말"을 暗示하고 title도 "안내"인데, 실제로는 **"죽음 준비" 카테고리 콘텐츠 피드**가 열린다. 사용자는 도움말/가이드를 기대하고 눌렀다가 무거운 주제의 글 목록을 만난다.
2. **왜 death_prep 고정인가 (High).** 글로벌 상단 아이콘이 특정 카테고리로 직행한다. 일반 맥락에서 갑작스럽고, 특히 첫 방문/비로그인에서 첫인상 부담이 크다.
3. **하단탭 커뮤니티와 기능 중복 (Med).** 도착 화면이 커뮤니티 탭과 사실상 같다(필터만 다름). 상단 아이콘만의 고유 가치가 없다.

### ✏️ "오늘 일기"
4. **불필요한 경유 리다이렉트 (Med).** `seed.html`을 한 번 로드한 뒤 곧바로 `diary-write.html`로 보낸다. 화면 깜빡임 + 뒤로가기 히스토리가 꼬인다(뒤로 누르면 seed로 튕김). → `diary-write.html` 직접 링크가 맞다.
5. **맥락 무시 (Med).** 모든 화면에서 ✏️ = "일기 쓰기"로 고정. 커뮤니티/콘텐츠 화면에서는 "글/포스트 쓰기"를 기대할 수 있다. 단일 기본값이 좁다.
6. **의미 모호 (Low).** 연필만으로 "일기"인지 "아무 글 쓰기"인지 구분 안 됨.

### IA(정보구조) 레벨
7. **상단 우측 자리의 정체성 부재 (High).** 한쪽은 특정 카테고리 딥링크(📚), 한쪽은 단일 작성 액션(✏️). 둘을 묶는 일관된 멘탈모델이 없다. 보통 이 자리는 검색/알림/저장/작성 중 하나의 일관된 역할을 가진다.
8. **작성 진입점 분산 (High).** 홈 ✏️(일기) · 라이프 FAB(일기/버킷) · 커뮤니티 FAB(콘텐츠/포스트, "어떤 글?" 시트) · 케어링 기록 버튼. 화면마다 "쓰기"가 다른 것을 의미해 멘탈모델이 흩어진다.

## 3. 제안 (재전략 후보 — 결정 전 옵션)

### A. 📚 자리의 재정의 (택1)
- **A1. 진짜 "가이드/안내":** 서비스 사용법 + 입문 큐레이션(죽음준비/케어링/라이프 시작하기) 한 페이지. 책 아이콘 의미에 부합.
- **A2. "내 서재/저장":** 북마크·내 글·내 기록 모아보기. 책 아이콘과 가장 자연스러움. 재방문 가치↑.
- **A3. 제거:** 하단탭 커뮤니티와 중복이면 과감히 정리하고 상단을 비운다(또는 검색으로 대체).

### B. ✏️ 자리의 재정의 (택1)
- **B1. 직접 연결 + 리다이렉트 제거:** `diary-write.html`로 바로. (최소 수정, 즉시 가능)
- **B2. 통합 작성 시트:** 탭하면 "무엇을 쓸까요?"(일기 · 버킷리스트 · 커뮤니티 글) 바텀시트. 전 화면 공통.
- **B3. 맥락형:** 화면별 기본 타입(홈=일기, 커뮤니티=포스트).

### C. 작성 IA 통일
- 전 화면 공통 "쓰기" 액션 1개 + 컨텍스트 시트(B2)로 분산을 수렴. FAB/상단/탭의 역할을 한 번 정리.

## 4. 전략 세션용 열린 질문
1. **상단 우측은 무엇을 위한 자리인가?** (검색 / 알림 / 저장(서재) / 작성 / 도움말 — 하나로 정해야)
2. **📚의 의도된 목적은?** death_prep 고정이 의도인가, 임시였나?
3. **작성(쓰기) IA를 하나로 통일할 것인가, 화면별 특화할 것인가?**
4. **비로그인 첫인상에서 무엇을 강조할 것인가?** (현재는 death_prep로 보내 부담 — 가치제안/온보딩이 먼저 아닌가)
5. 검색은 어디에? (지금 어디에도 전역 검색 진입점이 안 보임 — 콘텐츠가 쌓일수록 필요)

- ✏️ → `diary-write.html` 직접 링크(경유 리다이렉트 제거). *(문제 4)*
- 📚 title/aria를 실제 동작과 일치시키기(현재 "안내"인데 피드). 방향 정해지면 일괄.
