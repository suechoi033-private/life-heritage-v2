# 핸드오프 — "정보(info.html) 허브" 진입 경로 부재 (커뮤니티 재검토에 반드시 포함)

> 작성: 2026-06-07 (PE 세션) · **수신: 커뮤니티 전면 재검토 담당 세션**
> 상태: 🔴 미해결 — 이번 재검토에서 **반드시 함께 다뤄주세요.** 다른 세션이 커뮤니티(forest)를
> 동시에 작업 중이라 충돌을 피하려 이 세션에서는 nav/forest를 건드리지 않고 문서로만 남깁니다.

## 한 줄 요약

`info.html`(정보 허브 — "미리 · 곁에 · 떠난 뒤" 3시점 안내 콘텐츠)로 가는 **눈에 띄는 메인 진입
경로가 없다.** 진입점이 전부 작은 이모지 아이콘/칩이라 사용자가 못 찾는다. 커뮤니티 재검토 시
정보 허브의 위치·진입점을 함께 정리해야 한다.

## 사장님 제보 (실사용 경로)

> "정보로 가는 경로가 메인경로에서 안 보여. 마이 > 나의 삶 정리 > 여기서 404가 아닌 건 맨 위
> '사전연명의료의향서'뿐. 이걸 누르면 잇다에디터 공식 콘텐츠가 열리고, 거기서 뭘 누르니까
> 그제서야 정보 화면으로 감. 정작 '커뮤니티'를 눌러도 그 정보 화면은 안 보임."

즉 사용자가 정보 허브에 도달한 경로:
`마이(root) → 나의 삶 정리(self.html) → '사전연명의료의향서' 카드 → info/advance-directive.html
(또는 content-detail) → 본문 내 링크 → info.html`
— 우연히 발견한 5단계 우회 경로. 메인 내비게이션(하단 5탭)에서는 직접 닿을 수 없음.

## 현재 진입점 전수 조사 (전부 약함)

| 위치 | 형태 | 문제 |
|---|---|---|
| `index.html`(홈) 등 상단바 | `infoIcon: true` → 작은 📚 이모지 | 라벨 없음, 너무 작아 인지 안 됨 |
| `seed`(라이프)·`care`·`nest`(케어링)·`self` 상단바 | 동일 📚 | 동일 |
| `forest.html`(커뮤니티) | 카테고리 칩 줄 맨 앞 `📚 정보` 칩 1개 (`nav.js`엔 `infoIcon:false`) | 가로 스크롤 칩에 묻힘. 커뮤니티엔 상단 아이콘도 없음 |
| `onboarding.html` / `welcome.html` | 온보딩 중 1회성 링크 | 이후 재진입 불가 |
| `note/will.html`, `content-detail.html` | 본문/관련 링크 | 우연 의존 |

→ **하단 5탭(홈·라이프·케어링·커뮤니티·마이) 어디에도 "정보"가 1급 시민으로 없음.** 정보 허브는
사실상 고아 페이지에 가깝다.

## 왜 커뮤니티 재검토에서 다뤄야 하나

`info.html`은 **편집형 안내 콘텐츠**(사전연명의료의향서·상조 가입 가이드·가족 회의·보험연금
점검·디지털 정리 등)를 "미리/곁에/떠난 뒤" 시간축으로 묶은 허브다. 이는 커뮤니티(forest)의
콘텐츠 카테고리 체계와 **개념적으로 겹친다.** 두 콘텐츠 체계(정보 허브 vs 커뮤니티 카테고리)가
나란히 존재하면 사용자가 어디로 가야 할지 혼란스럽다. 커뮤니티 정보 구조를 다시 짜는 지금이
**정보 허브의 위치·진입점·중복을 함께 정리할 적기**다.

## 재검토 시 결정해야 할 것 (체크리스트)

- [ ] 정보 허브의 **정규 위치** 확정 — 커뮤니티 안으로 흡수할지, 별도 허브로 유지할지.
- [ ] **눈에 띄는 진입점** 마련 — 작은 이모지/칩이 아니라 라벨 있는 버튼/배너/섹션. 기본 랜딩(홈)과
      커뮤니티 양쪽에서 보이게.
- [ ] 정보 허브의 3시점(미리·곁에·떠난 뒤)과 **커뮤니티 카테고리의 중복 정리** — 한 콘텐츠가
      두 군데서 다르게 노출되지 않도록.
- [ ] `nav.js`의 📚 `infoIcon` 처리 일관성 — 켜고 끄는 페이지가 제각각(커뮤니티만 꺼져 있음).
      재검토 결론에 맞춰 통일.

## 함께 발견된 인접 이슈 (참고)

- **`마이 > 나의 삶 정리`의 404 데드엔드**: `self.html` 카드 중 `note/digital.html`·`funeral.html`·
  `messages.html`·`family-lock.html`은 **아직 페이지가 없어 404**. 현재 404 아닌 항목은 ①사전연명의료
  의향서(info/advance-directive) ②이번에 추가한 **의향서 방문 준비 체크리스트**(note/directive-checklist.html)뿐.
  → 미구현 카드는 "준비 중" 처리하거나 숨겨 404 데드엔드를 없애야 함(seed-16 웰다잉 가이드 메모에서도
  동일 우려 지적됨).

- `forest.html:430` — 커뮤니티의 `📚 정보` 칩
- `self.html` — 나의 삶 정리(카드 중 다수가 404)
