## 목표 구조 (모노레포)

```
life-heritage/            # 저장소 하나
├─ apps/
│  ├─ mutda/              # Flutter 앱 ① (우선 이식)
│  └─ itda/               # Flutter 앱 ② (케어링·정보·이후 디지털납골당)
├─ packages/
│  ├─ lh_ui/              # 디자인 토큰(에버그린 팔레트)·공용 위젯 — 원칙 #2·#4
│  ├─ lh_core/            # Supabase 인증·DB·푸시 클라이언트 — 원칙 #4
│  └─ lh_legal/           # 개인정보방침·약관 공통 화면 — 원칙 #3
├─ web/                   # 기존 정적 웹 (랜딩·마케팅·웹 폴백으로 존속)
└─ .github/workflows/     # 공통 배포 파이프라인 — 원칙 #1
```
멜로스(melos)로 패키지 관리. 앱 둘이 같은 packages를 쓰므로
창업자 4원칙(공통 배포·디자인 시스템·방침 템플릿·중복 금지)이 구조로 강제된다.
