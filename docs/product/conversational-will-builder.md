# 대화형 삶 정리 → AI 유언 초안 → (준비중) 디지털 공증

> 상태: **설계 확정(창업자 결정 반영) · 구현 대기**. 1차 빌드는 "케어링 업데이트 알림"이 먼저(2026-06-10).
> 관련: `note/will.html`(현 유언 메시지), `note/digital.html`(디지털 자산 노트), `onboarding.html`(재사용할 대화형 플로우),
> `docs/product/ai-vault-living-record.md`(상위 AI 보관함 설계).

## 1. 한 줄 정의
가벼운 질문 몇 개에 답하면(스킵 자유), 그 응답을 모아 **유언의 내용·구조(초안)**를 갖춰주고,
**그대로 따라 쓰는 자필 가이드**로 *법적 효력*까지 이어준다. 화상공증 연동은 "준비중".

## 2. ⚠️ 법적 전제 (반드시 지킬 것 — YMYL)
- **AI가 만든 인쇄·전자 텍스트는 그 자체로 유효한 유언이 아니다.** 한국 민법 제1066조(자필증서)는
  **전문·연월일·주소·성명을 유언자가 직접 손으로 쓰고 날인**해야 유효(한 글자라도 대필·인쇄면 무효).
  대법원도 전자문서는 서면 요건을 충족하지 못한다고 봄.
- 따라서 제품 카피는 **"유언장을 발급/생성"이라 단정하지 않는다.** 우리가 하는 일:
  ① 내용·구조 **초안**을 갖춰줌 → ② **자필로 따라 쓰는 가이드** 제공(효력은 사용자의 자필+날인에서 발생).
- **공정증서 유언**(공증인+증인 2명, 손글씨 불요)이 디지털 친화 경로 → **화상공증**(법무부 전자공증시스템
  enotary.moj.go.kr, '편리한 공증' 앱)으로 연결 가능. **단, 1차에서는 "준비중이에요"로만 표기**(파트너십·법무 감수 후 연동).

## 3. 창업자 결정 (2026-06-10)
- 질문은 **가볍게/진지하지 않게**. 단, **각 질문 하단에 회색 작은 글씨로 "이 답이 채우는 법적 요건"을 안내**
  → 사용자가 맥락을 알고 더 잘 답하게. (예: "누구에게 무엇을 남기고 싶어요?" 아래 회색: *상속인·재산·분배 요건을 채워요*)
- 출구는 **초안 + 자필 가이드까지**. 화상공증 연동은 **"준비중이에요"** 문구.
- 게이미피케이션은 **존엄하게**(헌장: 점수·랭킹·경쟁 금지). 진행은 "마음을 건네는 준비"가 차오르는 형태.

## 4. 질문 ↔ 법적 요건 매핑 (회색 힌트 포함)
| 가벼운 질문(화면) | 회색 힌트(하단) | 채우는 유언/상속 요건 |
|---|---|---|
| 누구에게 무엇을 남기고 싶어요? | *상속인·재산·분배를 정하는 부분이에요* | 상속인·수증자 지정 + 재산 특정 + 분배 |
| 꼭 따로 챙기고 싶은 사람이 있어요? | *특정유증·유류분과 관련돼요* | 특정유증 / 유류분 고려 |
| 이걸 누가 정리해 주면 좋겠어요? | *유언집행자를 정하는 부분이에요* | 유언집행자 지정 |
| 남기고 싶은 말이 있어요?(자유) | *마음을 담는 부분 — 법적 요건은 아니에요* | (정서적, 법적 필수 아님) |
| (자동 확인) 오늘 날짜·성함·주소 | *자필 유언의 필수 형식 요건이에요* | 연월일·성명·주소 자서 요건 |

## 5. "요건 충족도" 엔진
- 응답을 모아 **법적 필수 항목 충족률**을 백그라운드로 계산(사용자에겐 숫자%가 아니라 부드러운 상태로).
- 장기 미충족 시 푸시/인앱: *"○○님, 거의 다 왔어요. 3개만 더 답하면 유언장 형태가 완성돼요."*
  (1차에서 만든 `notifications` 인프라 재사용 가능 — type='will_nudge')

## 6. 출구 2개
1. **자필 가이드**: AI 초안을 보여주고 "이대로 손으로 따라 쓰세요" + 요건 체크리스트(전문 자서/연월일/주소/성명/날인).
2. **화상공증**: "준비중이에요" 안내 + 법무부 전자공증시스템 소개 링크(연동은 후속).

## 7. 데이터·기술
- **재사용**: `onboarding.html`의 단계형 플로우(진행바·단일/다중선택·자유입력·건너뛰기) 거의 그대로.
  `note/digital.html`의 "마음 한 줄" 톤도 차용.
- **AI 생성**: Edge Function `vault-will`(Claude) — 응답 JSON → 유언 초안 텍스트. `ANTHROPIC_API_KEY` 준비됨(미배포).
- **서버 저장 필요**: `note/will.html`은 현재 localStorage 전용. 알림·요건 충족도·AI를 쓰려면 서버 테이블로 승격.
  - 후보: `will_answers`(user_id, q_key, value, skipped, updated_at) + `will_drafts`(user_id, body, requirement_state jsonb, status).

## 8. 단계 (제안)
1. 대화형 UI 프로토타입(AI 없이) — 질문·회색 힌트·진행·스킵 + 서버 저장 + 요건 충족도.
2. `vault-will` Edge Function 배포 → 초안 생성 + 자필 가이드 화면.
3. 요건 미충족 nudge 알림(1차 notifications 재사용).
4. (후속) 화상공증 연동 — 법무 감수·파트너십 후.
