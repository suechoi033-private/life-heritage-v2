## 1. 한 줄 정의

**잇다는 사람들이 자기 죽음을 미리 대비하도록, 콘텐츠로 만나 도구로 준비하게 하는 서비스다.**

콘텐츠가 곧 (1) 고객 유입 채널, (2) 미션 실현 수단, (3) 자연스러운 수익 연결을 동시에 한다.

---

## 2. 문제

- 한국에서 죽음·임종 준비는 금기. 대부분 "닥쳐서야" 부랴부랴 처리한다.
- 부모를 보낸 사람은 "그때 너무 막막했다, 미리 알았으면" 후회한다.
- 정보가 흩어져 있고(요양·장례·상속·연명의료·디지털 자산), 누구에게 묻기도 어렵다.
- 형제 공동 돌봄 상황에서는 정보가 카톡에 흘러가 갈등·후회를 만든다.

## 3. 솔루션

**lifecycle 콘텐츠 허브 + 사전 준비 도구 앱.**
- 검색·SNS로 들어온 사람이 자기 상황(돌봄/사별/준비)에 맞는 글을 만난다.
- 글 끝에서 자연스럽게 잇다의 도구(체크리스트·일지·사전연명 등록·디지털 vault·가족에게 남기는 말)로 연결된다.
- 한 사람을 30년 lifecycle(부모 돌봄 → 사별 → 자기 준비 → 회고 → 다음 세대)로 동반한다.

## 4. 타깃 (페르소나 2개 동시)

1. **입구 페르소나**: 40~60대 직장인, 형제 2~4명 중 한 명, 부모가 막 요양시설 입소·진단 직후
2. **미션 핵심 페르소나**: 50~60대, 부모 사별 후 1~3년, "내 자식들엔 안 그러게 하고 싶다"

## 5. 시장

- 일본 終活·장례 시장 약 1.5~1.8조 엔(15~18조 원). 한국은 고령화가 일본을 10~15년 후행 → 형성기 진입(early).
- 한국 50~70대 약 1,800만 명, 부모 사별 경험자 약 900만 명.
- "모수 게임"이 아니라 **ARPU 게임** — 5,000명 결제자 + lead gen으로 수십억 매출 가능.

## 6. 비즈니스 모델 (수익 다리 4개 — 단일 의존 금지)

| 다리 | 형태 | 단가 |
|---|---|---|
| 구독 | 월 9,900~19,900원 (가족 단위) | 반복 |
| 도구 유료화 | 책 출판, 프리미엄 체크리스트 | 건당 5~50만 원 |
| 매칭(lead gen) | 요양원·간병·상조·법무·세무 | 건당 30~300만 원 |
| B2B | 기업 HR·EAP(임직원 grief 지원) | 계약형 |

목표 ARPU: 월 30,000~50,000원(구독 + 매칭 합산).

## 7. 차별점 (벤치마킹 기반)

미국 Modern Loss(콘텐츠 강·도구 없음)와 일본 終活ねっと(SEO 매칭·신뢰 약함)가 **동시에 놓친 빈 자리**:
1. 사전 준비(advance) 카테고리 owner
2. interactive 도구·앱 (콘텐츠 → 행동)
3. B2B(HR·EAP)
4. lasting community (익명 peer circle)
5. 한국 의례 캘린더(49재·100일·1주기·명절 제사) native

## 8. 절대 피할 함정 (실패 사례에서)

- YMYL 양산 금지 → 변호사·세무사·장례지도사·의사·심리사 監修 명단 visible
- 매칭 단일 수익 금지 → 다리 4개 동시
- 콘텐츠/도구 분리 금지 → single domain 유지
- VC 받고 paid 광고 ramp 금지 → organic 우선
- 수익 모델 후행 설계 금지 → 출발부터 매출 3축 명확

## 9. Go-to-Market (콘텐츠 엔진)

- 시드 콘텐츠 30~50개(잇다 직접 작성) → SEO·SNS 자연 유입
- 가이드(SEO 입구) + 에세이(머무는 이유) 균형, 각 글에 도구 CTA
- 창립자 narrative + 신문 시그니처 feature 1편(6개월 내)
- **살아서 하는 장례식** 오프라인 이벤트는 베이스 확보 후 PR 카드로 (12개월 후)

## 10. Phase 로드맵

| Phase | 기간 | 목표 |
|---|---|---|
| 0. 시드 콘텐츠 | 0~3개월 | 글 30~50개, SEO 인덱싱 |
| 1. 베이스 | 3~9개월 | 가입 2~5천 |
| 2. UGC 활성화 | 6~12개월 | 사용자 글 > 잇다 글 |
| 3. PR 폭탄 | 12개월+ | 살아서 하는 장례식 1호 |
| 4. 미션 본진 | PR 후 | 사전연명·상속·디지털 결제 본격화 |

## 11. 지금의 검증 계획 (90일)

1. **프로덕트 정리**: 미션에 맞게 메뉴·기능 정리(뺄 것/넣을 것)
2. **지인 20명 테스트**: 페르소나인 지인 → 인터뷰 / 비페르소나 지인 → 사용성 테스트
3. **콘텐츠 병렬**: 시드 콘텐츠 #1·#10부터 작성 시작
4. 핵심 질문: "페르소나가 실제 존재하나 / 도구를 끝까지 쓰나 / 돈 낼 의향 있나"

## 12. 리스크

- 페르소나 가설이 틀릴 위험 → 인터뷰로 조기 검증
- "죽음" 주제 진입 장벽 → 콘텐츠 톤("위로 말고, 같이")으로 완화
- 지인 샘플 편향 → 사용성/시장 검증 분리, 추후 모르는 사용자로 재검증
- YMYL 신뢰 → 監修 체계 처음부터
