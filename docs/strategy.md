> Claude와의 전략 논의 정리. 결정된 것 · 검토 중인 것 · 다음에 할 일.

## 1. 미션 (확정)

**"사람들이 자기 죽음을 미리 대비하게 한다."**

- 본 미션은 잇다의 north star.
- 모든 메뉴·기능·콘텐츠는 이 미션에 기여하는지 기준으로 판단.
- 일본 슈카츠(終活) 시장이 1조 엔 규모. 한국은 아직 초기 = 카테고리를 만들 기회.

## 2. 비즈니스 모델 (확정)

**라이프이벤트형 SaaS + Lifecycle Lead Generation**

- "모수 vs ARPU" 관점 → **잇다는 ARPU 게임**. 모수 작아도 사업 가능.
- 5,000명 결제자 + lead gen이면 수십억 사업.

### 매출 구조

| 분류 | 형태 | 예상 단가 |
|---|---|---|
| 본업 | 월 구독 (가족 단위) | 월 9,900 ~ 19,900원 |
| 부가 | 책 출판 | 건당 30~50만 원 |
| Lead gen | 요양원·간병 매칭 | 건당 30~50만 |
| Lead gen | 상조·장례·유품정리·법무사 | 건당 30~100만 |
| Lead gen | 변호사(상속)·세무사·신탁 | 건당 100~300만 |

**Lead gen은 광고가 아님.** 사용자에게 "지금 필요한 다음 단계"를 정확하게 매칭 — 사용자도 파트너도 둘 다 고마워하는 구조.

### 목표 ARPU
- 월 30,000 ~ 50,000원 (구독 + 매칭 수수료 합산)
- 비교: 카카오톡 1,000원 / 노션 8,000원 / Trust&Will 25,000원

## 3. 제품 구조 (확정 = Lifecycle Funnel)

한 사람을 30년 동안 들고 가는 lifecycle SaaS.

```
[1. 케어링 입구]    [2. 사별·정리]      [3. 자기 준비]      [4. 회고·남김]     [5. 다음 세대]
 형제 공동 돌봄      장례·상속·유품       "나도 미리"          손주에게 남기는       가족이 잇다에서
 부모 요양시설       가족이 헤맴          사전연명·디지털       회고록·책            추모·절차
 40~60대 직장인      같은 사람,           같은 사람,            같은 사람,           자녀가 새 입구로
 진단·이사 직후      6~12개월 후          1~3년 후              5~10년 후            (1단계로 회귀)
```

### Transition Trigger (가장 중요)

| 전환 | trigger 이벤트 | 잇다가 해야 할 nudge |
|---|---|---|
| 1→2 | 부모님 사별 | "장례 후 90일 안에 해야 할 7가지" 자동 안내 |
| 2→3 | 사별 후 6~12개월 | "그때 가장 막막했던 거 기억하세요? 나의 가족을 위해 미리…" |
| 3→4 | 환갑·은퇴·손주 출생 | "이 시간을 글로 남기시겠어요?" |
| 4→5 | 본인 사망 | 가족이 잇다 계정 인계 → 미리 작성된 가이드로 절차 진행 |

**1→2와 2→3 사이가 가장 약한 고리이자 가장 큰 기회.** 여기 잘하면 한국 최초의 "사전준비 카테고리"를 만들 수 있음.

### 메뉴 재설계안 (2차 — 커뮤니티가 거의 홈)

기존 B안(4메뉴) 폐기. 콘텐츠 중심 구조로 변경.

```
[홈] = 콘텐츠 피드 (오늘의 글 + 내 단계에 맞는 추천 글)
  │
  ├─ [읽기] 콘텐츠 (커뮤니티의 본체)
  │
  ├─ [쓰기] 내 경험·생각 글로 남기기
  │
  ├─ [도구] 사전연명·일지·체크리스트·책 출판
  │
  └─ [내 단계] 지금 내 lifecycle 위치 + 다음 단계
```

홈에 들어오면 가장 먼저 보이는 것 = "오늘의 글 3개". 무거운 메뉴 4개 X. **글 하나하나가 진입로.**

### Phase별 KPI (확정)

| Phase | 기간 | 목표 | KPI |
|---|---|---|---|
| 0. 시드 콘텐츠 | 0~3개월 | 30~50개 글 발행, SEO 인덱싱 시작 | 글 수 |
| 1. 베이스 만들기 | 3~9개월 | SEO·SNS 유입으로 가입 2~5천 | 가입자, 글당 평균 체류시간 |
| 2. UGC 활성화 | 6~12개월 | 사용자 글이 잇다 글보다 많아짐 | UGC 비율, 재방문율 |
| 3. PR 폭탄 | 12개월 + | 살아서 하는 장례식 1호 영상화 → 방송·SNS | 노출 수, 가입 폭증 |
| 4. 미션 본진 | PR 후 | 사전연명·상속·디지털 정리 결제 본격 시작 | ARPU, lead gen 매출 |

## 4. 톤·접근 + 커뮤니티 재정의 (2차 논의 결과)

### 문제
한국에서 죽음은 금기. 정공법으로 가면 아무도 안 들어옴.

### 핵심 통찰 (2차 논의)
- "챌린지"라는 표현보다 **"콘텐츠 시리즈"**가 맞는 정체성.
- 이런 콘텐츠를 모으고 사용자가 자기 글·경험·생각을 나눌 수 있는 지면 = **커뮤니티**.
- 그래서 커뮤니티는 게시판이 아니라 **lifecycle 콘텐츠 허브**.
- 살아서 하는 장례식 PR은 **베이스(가입자) 만든 후로 아껴둠**. 빈 앱에 데려와봤자 다시 안 옴.

### 커뮤니티의 정체성

> **잇다 커뮤니티 = "인생 단계별 경험·생각·실용 정보를 글로 남기고, 같은 단계의 다음 사람이 그 글로 길을 찾는 지면"**

비교 모델: 미국 *Modern Loss*, 일본 *終活ねっと*. 한국에는 아직 없음.

### 콘텐츠 카테고리

| 카테고리 | 예시 | 누가 읽나 (lifecycle 단계) |
|---|---|---|
| 가족에게 남기는 말 | "내가 떠난 후 가족이 알아야 할 7가지" | 3 (자기 준비) |
| 사별 후 90일 가이드 | "부모님을 갑자기 잃은 40대가 해야 할 일" | 2 (사별 직후) |
| 함께 돌보는 법 | "치매 어머니 형제와 분담한 방법" | 1 (케어링) |
| 마지막 대화 | "아빠에게 처음 물어본 질문" | 1~2 |
| 살아서 정리한 사람들 | "60세에 디지털 자산 정리했더니" | 3~4 |
| 회고·후회 | "30년 직장 끝나고 처음 한 후회" | 4 |

각 글은 동시에 3가지 역할:
1. **SEO 유입** (네이버·구글에서 자연 도달)
2. **공감·신뢰 형성**
3. **자연스러운 lead gen** (글 끝에 "관련 잇다 도구" CTA)

### 콘텐츠 생산 모델: Hybrid (확정)

| 옵션 | 결정 |
|---|---|
| A. 잇다 큐레이션 단독 | ✗ 양 안 나오고 비용 큼 |
| B. 사용자 UGC 단독 | ✗ 초기 빈 앱 문제 |
| **C. Hybrid (잇다 시드 30~50개 + UGC 확장)** | ✓ 확정 |

- 처음 3개월: 잇다가 시드 콘텐츠 30~50개 작성
- 모든 시드 글에 "당신은 어땠나요?" CTA → 답글이 자연스럽게 새 콘텐츠
- 6개월 후 목표: 사용자 글 > 잇다 글
- 모든 사용자 글은 1차 review 후 메인 노출

### 시드 콘텐츠 30개 분배 (초안)

```
가족에게 남기는 말 ......... 8개
사별 후 90일 가이드 ........ 8개  (실용성 최고, SEO 최강)
함께 돌보는 법 ............. 6개
마지막 대화 ............... 4개
살아서 정리한 사람들 ........ 4개
```

각 글 약 1,500~2,500자, 마지막에 "잇다에서 할 수 있는 것" CTA.

### 살아서 하는 장례식 (PR 카드, 12개월 후)
- 월 1명 선발, 비용 200~500만 원 (장소·꽃·영상·MC)
- 가치: 영상·기사 1건 → 가입자 수천 명 유입
- 비교: 일본 NHK 다큐 1편이 슈카츠 시장을 만든 사례
- **12개월 전에는 콘텐츠로만 등장** (글 형태로 호기심 쌓기)

## 5. 페르소나 (작업 중)

### Lifecycle 입구 페르소나 (1단계)

> 잇다는 **부모님이 막 요양원·요양병원으로 옮겨진 40~60대 직장인, 형제 2~4명 중 한 명인 사람**이
> **카톡 그룹채팅에 흩어지고 흘러가버리는 면회 메모·의사 소견·약 변경·결정 사항**을
> **형제 모두가 같은 페이지에서 보고, 6개월 뒤 "그때 누가 뭐라 했더라" 후회·갈등 없이 이어갈 수 있는 공식 기록**으로 만들기 위해
> **매주 면회 후 일지를 남기고 형제와 공유**하는 앱이다.

### 미션 핵심 페르소나 (3단계)
- 50~60대, 부모 사별 후 1~3년
- 트리거: "그때 막막했다, 내 자식들엔 안 그러게 하고 싶다"
- 핵심 행동: 사전연명의료의향서 → 보험·연금 정리 → 디지털 계정 정리 → 가족에게 위치 알림

### 회고 페르소나 (4단계)
- 60대 은퇴 직후
- 트리거: 환갑·은퇴·손주 출생
- 핵심 행동: 매일 1개 질문 답변 → 1년 후 회고록 자동 정리

## 6. 다음 액션 (논의 후 진행)

### 지금 진행 중인 논의
**시드 콘텐츠 첫 10개를 같이 정하기.** 제목·타깃 lifecycle 단계·SEO 키워드까지 잡으면 누가 쓸지, 어디 들어갈지, 어떤 도구로 연결될지가 다 풀림.

### 논의 후 진행할 액션 4가지 (저장)

#### A. 인터뷰 질문지 만들기
- **대상**: 형제 공동 돌봄자 5명, 사별 경험자 5명
- **목적**: 1단계 입구 가설 + 2→3 lifecycle 다리 가설 검증
- **시간**: 30분/명, 총 5시간
- **결과물**: 질문지(.md) + 인터뷰 결과 정리 템플릿

#### B. 홈 (index.html) 재구성
- 현재: 라이프·케어링·커뮤니티 3개 동등 노출
- 변경: **콘텐츠 피드 중심**. 오늘의 글 + 내 단계에 맞는 추천 글.
- 단, 시드 콘텐츠 10개 확정 후 진행

#### C. Lifecycle 데이터 모델 설계
- 사용자가 단계 1→5로 이동하는 schema
- "현재 단계", "다음 단계 예상 시점", "transition trigger 이벤트"
- 콘텐츠 글에 lifecycle 단계 태그 (추천에 사용)
- Supabase schema 변경 포함

#### D. 시드 콘텐츠 30~50개 작성 계획
- 누가 쓸지 결정 (사용자 본인 / 외주 / 에디터 고용)
- 발행 일정 (주 2~3개 × 3개월)
- 글 템플릿 (제목·서두·본문·CTA 구조)
- 첫 10개는 지금 논의에서 결정

## 8. 벤치마킹 종합 — Modern Loss + 終活ねっと

> 상세 보고서: `docs/benchmark/modern-loss.md`, `docs/benchmark/shukatsu-net.md`

### 한 줄 통합
- **Modern Loss (US)**: 콘텐츠+책+강연으로 brand 만들었지만 **도구·B2B 없어 plateau**
- **終活ねっと (JP)**: SEO+매칭으로 빠르게 성장했지만 **신뢰·다각화 없어 6년 만에 소멸**
- **잇다의 자리** = 둘이 동시에 놓친 빈 자리 — **감성 콘텐츠 + 사전 준비 도구 + 다각화 수익**

### 두 회사 정반대 비교 → 잇다의 답

| 차원 | Modern Loss | 終活ねっと | 잇다가 가야 할 길 |
|---|---|---|---|
| 콘텐츠 형식 | personal essay (감성) | SEO 가이드 (정보) | **둘 다** — 가이드 입구 + 에세이 머무는 이유 |
| 콘텐츠 생산 | 잡지형 pitch-edit | 학생 양산, 監修 없음 | **잡지형 + 監修 명단 visible + AI 초안 + 전문가 sign-off** |
| 수익 | 책·강연·구독·grant | 매칭 단일 | **다리 4개: 구독 + 도구 + 매칭 + B2B(EAP·HR)** |
| 도구·앱 | 거의 없음 | 매칭 폼만 | **lifecycle native app — 가장 큰 차별** |
| 사후/사전 | 사후 grief | 사후 葬儀 매칭 | **사전 준비 (advance) — 빈 시장** |
| 커뮤니티 | 약함 (Substack thread) | 없음 | **앱 내 anonymous peer circle** |
| PR | NYT 1번이 평생 카피 | TV CM ramp가 destroy 시작 | **창립자 narrative + 한겨레/중앙 시그니처 feature** |
| 톤 | "Bit of attitude; zero platitude" | 정보형 (감정 zero) | **"위로 말고, 같이"** |

### 둘이 동시에 놓친 5가지 = 잇다의 white space
1. **사전 준비(advance) 카테고리 owner** — Modern Loss는 사후만, 終活ねっと는 매칭만
2. **interactive 도구·앱** — 콘텐츠를 체크리스트·vault·작성기로
3. **B2B (HR·EAP·기업 복지)** — 한국 대기업 grief support 시점
4. **lasting community** — 익명 peer circle
5. **한국 의례 캘린더** — 49재·100일·1주기·명절 제사 native

### 강력하게 피해야 할 함정
1. YMYL 양산 (終活ねっと) → 監修자 명단 visible
2. 콘텐츠 only / product 없음 (Modern Loss) → 콘텐츠는 입구, product가 본체
3. 매칭 단일 수익 (終活ねっと) → 다리 4개 동시
4. 콘텐츠와 도구 분리 (終活ねっと 2021) → single domain 유지
5. VC 받고 paid ramp (終活ねっと 2019) → organic 우선
6. 수익 모델 후행 설계 (Modern Loss 본인 자백) → 런칭 시점 매출 3축

## 9. 시드 콘텐츠 첫 10개 (벤치마킹 적용)

가이드 5편 (終活ねっと식 SEO 입구) + 에세이 4편 (Modern Loss식 머무는 이유) + 창립자 narrative 1편

| # | 제목 | 형식 | 길이 | lifecycle | 검색어 / 유입 | 잇다 도구 CTA |
