## 1. 의제 통합 표 — 사장님 3개 × PE 5개 어떻게 얽히는가

| 사장님 의제 | 연동 PE 게이트 | 얽힘 한 줄 |
|---|---|---|
| **A. 첫 화면 이원화 (비회원 = 1분 점검만 / 회원 = 지금 홈)** | **G1** (게이트 경로 A/B/C), **G5** (Supabase 동기화) | A를 채택하면 비회원 첫 화면이 사실상 G1 게이트 자체가 된다. 비회원이 답한 7문항을 가입 후 잃지 않으려면 G5(Supabase 동기화)가 같이 결정돼야 함. |
| **B. 모바일 지면 우선순위 (정답은 없어요 → 커뮤니티 → 요즘 잇다에선)** | **G1**, **G3** (welcome.html 처리), **G4** (login next) | 비회원에게 첫 fold에 무엇을 보이는가는 G1 게이트 경로의 입구 카피이자, G3 (옛 welcome.html 잔재)이 살아 있으면 모바일 지면 메시지가 분기되어 일관성 깨짐. 로그인 동선(G4)도 첫 fold 동선과 한 묶음. |
| **C. "정답은 없어요" 카드 정체 (회원 엔게이지 / 비회원 자기성찰 질문)** | **G1**, **G2** (emailRedirectTo), **G5** | C는 본질적으로 "한 카드가 두 페르소나에게 다른 의미로 쓰이고 있다"는 문제. A로 분리하면 자동 해소되지만, 회원 자리에 어떤 엔게이지가 들어갈지(=오늘 잇고? 다른 카드?)는 G2·G5로 받은 stage 데이터가 있어야 개인화 가능. |

**핵심 결합점**: A·B·C 세 개가 결국 G1(게이트 경로)·G5(답 동기화)·G3(옛 흐름 정리) 위에서만 일관되게 풀린다. 분리 결정 금지.

### 2.4 "정답은 없어요" 카피의 출처
- `index.html` L391 한 곳. 비회원 카드 eyebrow에만 존재. 회원 카드에는 없음(회원 카드는 `🕯 오늘 잇고`).
- 즉 "정답은 없어요"는 **이미 비회원 전용**. 사장님의 "회원의 오늘 첫 행동 자리"는 정확히 말하면 회원 카드(`오늘 잇고`)이고, "정답은 없어요"는 비회원 카드. 두 카드가 같은 자리(`card-seed`)·같은 그라데이션·같은 모양을 쓰므로 시각상 혼동된다. 사장님 직관의 근거는 그것.

### 2.5 게이트 / 인증 잔재 (PE QA 인용)
- `signup.html`은 `?next=` 받음, `safeNext` 통과. `login.html`은 next 미수용(G4 미해결).
- `auth.js` `emailRedirectTo` = `welcome.html` 고정 (G2 미해결).
- `welcome.html`은 옛 흐름 — 신 `onboarding.html`과 데이터 분리(G3).
- 비회원 onboarding 답은 localStorage 단독, Supabase 미동기화(G5).

## 3. 페르소나 × 첫 화면 동작 매트릭스

각 셀: `첫 보임 / 첫 행동 / 다음 행동 / 어색함(○매끄 △주의 ✕어색)`

| 페르소나 | 신규 비회원 (첫 진입) | 재방문 비회원 (localStorage 답 있음) | 로그인 회원·미관여 (가입 후 첫 진입) | 로그인 회원·관여 (재방문) |
|---|---|---|---|---|
| **① 부모 케어링 40·50대** | 정답은 없어요 카드 · 시작 → onboarding · 결과 → 게이트 / **△** "1분"이라는 가벼움이 부모 케어링자의 무거움과 결이 어긋남 | 같은 카드, 새로 시작? 이어서? 모호 / **✕** 코드상 같은 화면, 재진입 시그널 0 | 오늘 잇고 카드 · 답변 남기기 → ask · 첫 답 / **△** 가입 직후 onboarding 답이 회원 홈에 안 반영(G5 미해결) | 오늘 잇고 · 매일 짧은 답 / **○** 자연스러움 |
| **② 사별 후 자기준비 50대+** | 같은 카드 · "정답은 없어요"의 정답이라는 어휘 자체가 사별 직후엔 자극적 / **✕** 어휘 부적합 (헌장 점검 항목) | 답 보존되나 결과 재진입 동선 없음 / **✕** 5/30 결정한 lifecycle 분기 결과로 도달은 가능하나 UI 큐 없음 | 가입 후 회원 홈에 `장례희망` 카드가 두 번째 — bereaved 페르소나에 직격이지만 가입 직후엔 무거움 / **△** | 오늘 잇고가 bereaved에게 매일 적합한지 불확실 / **△** 콘텐츠 차원 |
| **③ 비혼·자기 죽음 준비 20·40대 여성** | 정답은 없어요 카드 → onboarding → explorer/self_prep stage 가능 / **○** 본인 페르소나(활성 시장)에 가장 잘 맞음 | 답 보존, 재진입 자연스러움 / **△** "이어서" 동선 부재만 보완하면 ○ | 오늘 잇고 → ask → 짧은 답 / **○** 활성 시장 핵심 페르소나로서 자연스러움 | 오늘 잇고 매일 + 피드 회유 / **○** v3 비즈니스 모델의 활성 레이어 그 자체 |

**가장 어색한 셀 2개 (우선 처치):**
1. **② × 신규 비회원**: "1분 · 정답은 없어요"라는 어휘가 사별 직후 페르소나에 자극적·경박. 헌장 톤 점검 1순위.
2. **① × 재방문 비회원**: 같은 화면이 재진입 시그널 없이 두 번 나옴. 1번 했던 사람이 어디서 이어가는지 모름. (G5 동기화 미해결과 직결.)

## 4. 옵션 비교 — 홈 구성 × 게이트 경로

### 4.1 홈 구성 옵션 (H1~H3)

| 옵션 | 비회원 첫 fold | 비회원 below the fold | 회원 첫 fold | 회원 below the fold |
|---|---|---|---|---|
| **H1 — 현행 유지 (사장님 직관 무시)** | 정답은 없어요(1분) + 커뮤니티 둘러보기 + 로그인 줄 | 요즘, 잇다에선(피드) | 오늘 잇고 + 장례희망 | 피드 |
| **H2 — 사장님 A안 강 (비회원=1분 only)** | 정답은 없어요(전체화면 큰 카드) — 다른 모든 글 제거 | (없음) 또는 "둘러보기" 매우 작게 | 현행 유지(오늘 잇고 + 장례희망 + 피드) | — |
| **H3 — 비회원 first-fold 재구성, below the fold는 콘텐츠 유지** | (상단) 잔잔한 인사 + 진단 입구 카드(어휘 톤 다운) | (중간) 커뮤니티 둘러보기 카드 (작게) → "요즘, 잇다에선" 피드 | 현행 유지 + stage 기반 카드 1개 개인화 자리 추가 | 피드(이웃 글 섞기 검토) |

### 4.2 게이트 경로 옵션 (PE QA 3.1 인용 — A/B/C)

PE는 **A+B 혼합**(현행 + CTA 시그널 강화) 권장, 이 라운드에 P1 적용 완료. 본 문서는 그 결정 유지를 전제로 한다.

### 4.3 (홈 × 게이트) 매트릭스 — 평가축 6개

| 조합 | (a) 모바일 first-fold | (b) acquisition 퍼널 | (c) D7 인지부조화 재발 | (d) G1~G5 일관성 | (e) 구현 비용 | (f) 헌장 일관성 |
|---|---|---|---|---|---|---|
| **H1 + A+B 혼합** (현행) | 1분 진단 + 커뮤니티 = 두 입구가 경쟁 | 분산 | 낮음(분기 트리로 해소) | G5 동기화 안 하면 재방문자 답 잃음 | 0 | "정답은 없어요" 자극·경박 위험 △ |
| **H2 + A+B 혼합** (사장님 A안 강) | 1분만 — 깔끔, 강력 / 그러나 콘텐츠 없는 사이트 인상 | 진단 완료율 ↑ / 진단 회피자(둘러보기 의향) 즉시 이탈 | **중**: 비회원이 "이 사이트가 뭐 하는 곳"인지 모른 채 7문항 받으면 D7 변종 재발 — "이거 왜 답해야 하지" | G1과 정합, G5 동기화 필수(없으면 모든 게 사라짐) | 중(`index.html` 분기 + 이웃 글 피드를 비회원에게 안 보이는 결정 동반) | **위험**: 들이대는 느낌. 헌장 "조용함" 위배 가능. 사장님 본인이 "비회원에게 1분 점검을 들이대는 게 애매하다"고 직관 |
| **H3 + A+B 혼합** (재구성) | 톤 다운한 진단 입구 + 잔잔한 콘텐츠 노출 (잇다에선 = 이웃 글 + 에디터 글 혼합) | 진단 입구 + 콘텐츠 큐레이션 = 두 입구 공존, 콘텐츠로 머무는 이유까지 동시 제공 (Modern Loss·슈카츠넷 동시 회피) | 낮음 | G5 동기화 필요, G3 정리 필요 | 중–상(피드 쿼리 확장, 카피 재작성) | **부합**: 헌장 톤·v3 활성 레이어 양립 |

### 4.4 평가

- **H1**: 사장님 직관(이미 "이게 의미 있을까" 자체 의심)이 H1을 거부. 무엇보다 ② × 신규 비회원 ✕ 셀이 그대로 남음.
- **H2**: 강력하지만 헌장 톤과 충돌. 비회원에게 "들이대는" 인상이 잇다 정체성을 깎는다. 또한 비회원이 콘텐츠를 본 적도 없는 상태에서 7문항 응답 → D7 변종(왜 답해야 하지) 재발 가능.
- **H3**: 톤 다운한 진단 입구 + 콘텐츠 큐레이션 공존. v3 사업계획서의 **양면 시장(20-40대 활성 × 30-60대 수익)** 구조와 가장 정합. 모바일 first-fold에 잔잔한 노출로 "사람들이 머무는 자리"를 만든다 = ARPU/LTV 동력.

## 5. 추천 — H3 + (G1: A+B 혼합 유지) + G2~G5 일괄 도입

**핵심 사유 3줄:**
1. 사장님 직관("정답은 없어요를 비회원에 들이대는 게 애매")과 헌장(조용함·존엄)·v3 활성 레이어 모델이 H3에서만 동시에 성립한다.
2. 모바일 first-fold는 v3 ARPU/LTV의 진짜 엔진 — 그러나 그 엔진은 "전환 광고"가 아니라 **콘텐츠 큐레이션 + 잔잔한 진단 입구의 공존**이라는 retention 자산의 형태로 작동해야 헌장과 양립한다. 사장님 통찰(모바일 지면=돈)을 "톤 다운한 retention 큐레이션"으로 받아라.
3. H3의 비회원 첫 fold가 의미 있으려면 답 손실 0 (G5)·옛 흐름 정리(G3)·login next(G4)·인증 redirect(G2)가 같이 풀려야 한다. 분리하면 어디선가 페르소나 무덤 생긴다.

**기준 카피 후보 (카피라이터 라운드로 위임 전제):**
- 비회원 1분 카드 어휘 후보:
  - (a) `잠깐, 지금 어디쯤이세요?` — 잔잔, 톤 다운
  - (b) `오늘 마음이 머무는 자리, 같이 짚어볼게요` — onboarding 결과 어휘("자리")와 일관
  - (c) `오늘은 어디쯤 와 계세요?` — 가장 무난, 헌장 톤 정합
  - **제외 권장**: `정답은 없어요` — 표현 자체가 "정답"을 환기시켜 부담을 역설로 만든다. 사별 직후 페르소나엔 자극적.

## 6. 사장님 결정 카드 (7개)

> 모든 카드는 **함께 결정**한다. 분리 결정 금지(의제 간 모순 발생).

### **D1. 비회원 첫 화면 구성**
- **결정**: H1(현행 유지) / H2(1분 only 강제) / **H3(톤 다운 진단 + 콘텐츠 공존)**
- **추천**: H3
- **영향**: 사장님 A안의 직관을 살리되 헌장 톤·v3 활성 레이어와 양립. ② 페르소나의 ✕ 셀 해소.

### **D2. 회원 첫 화면의 "오늘 첫 행동" 자리**
- **결정**: 현행 `오늘 잇고` 유지 / stage 기반 개인화 자리 추가 / 비회원과 시각 분리(다른 색·다른 카드 컴포넌트)
- **추천**: 현행 유지 + **비회원/회원 카드 시각 분리**(같은 `card-seed` 그라데이션 공유로 인한 혼동 제거. 회원만 그라데이션, 비회원은 잔잔한 솔리드 톤 등 — 카피라이터+PE 라운드 위임).
- **영향**: C 의제(카드 정체 모호) 시각 차원 해소.

### **D3. 모바일 first-fold "이웃 글"의 정의**
- **결정**: 현재 `listHomeFeed`는 **에디터 글만**(`author_type='official'`). 사장님의 "이웃 글"이 (a) 에디터 글을 이웃 톤으로 보이게 하는 시각 처리 / (b) 사용자 글 일부를 홈에 노출(필터·감수 룰 필요) / (c) 둘 다 점진 도입
- **추천**: 12개월 활성 검증 전엔 **(a) 시각·언어 처리만** — 사용자 글 노출은 헌장 YMYL·과시 금지·운영 비용 측면에서 6개월+ 후행. v3 가설 a/b/c 검증 후.
- **영향**: 모바일 지면 = 돈이라는 사장님 통찰을, organic 콘텐츠 자산 축적으로 받는다. 광고적 전환 X.

### **D4. G1 (게이트 경로)**
- **결정**: A(현행) / B(CTA→가입 직행) / C(1회 무가입) / **A+B 혼합**
- **추천**: A+B 혼합 (PE가 이 라운드 P1으로 이미 적용. 본 문서는 추인.)
- **영향**: 미러링 클라이맥스 보존 + CTA 시각 신호 강화.

### **D5. G2 (emailRedirectTo) + G3 (welcome.html)**
- **결정**: G2 = `signup.html`의 next를 `emailRedirectTo`에 동봉 / G3 = `welcome.html`을 index.html로 리다이렉트
- **추천**: 둘 다 도입. 분리 도입 금지(welcome.html 살아있으면 emailRedirectTo 수정해도 잔재 충돌).
- **영향**: 이메일 인증 환경에서 이중 온보딩 폭발 방지. one-way door는 아니지만 PE가 함께 처리.

### **D6. G4 (login.html next 보존)**
- **결정**: 도입 / 미도입
- **추천**: **도입**(signup의 `safeNext` 정규식 재사용).
- **영향**: 기존 회원이 게이트 → signup → "이미 계정 있어요" 클릭 시 결과·답 손실 방지.

### **D7. G5 (Supabase 답 동기화)**
- **결정**: 도입(`profile.onboarding_stage` + `answers` JSONB) / 미도입(localStorage 단독)
- **추천**: **도입**. 단 DB 스키마 변경은 **one-way door** — 창업자 승인 + 백업 후 PE 실행.
- **영향**: H3가 의미 있으려면 필수. 답 동기화 없으면 ① × 재방문 비회원 ✕ 셀 그대로. v3 stage 기반 홈 카드 개인화의 기초.
