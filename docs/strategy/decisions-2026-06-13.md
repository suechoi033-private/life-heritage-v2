## 2. 보류된 결정 (1개)

### D5. 베타 약속 정의 — **보류**

원안 추천: 무료 12개월·캡 50명·정식 1년 무료 (one-way door).
**사장님 응답**: *"구독료 형태로는 못 갈 것 같어."*

### 즉시 처리
- **D3·D4 마케터·PE 위임에 반영**: 카드뉴스 CTA와 `beta.html` 모두 **가격·기간·캡 약속 금지**.
- **무약속 톤** 채택 — 정체성·관계만 약속(예: "잇다 베타 사용자로 함께해 주실래요?", "피드백 채널이 열려 있어요").

## 3. 큰 신호 — 별도 라운드 트리거

"구독료 형태로는 못 갈 것 같어"는 단순 베타 약속 보류가 아니라 **v3 사업계획서의 BM 본질에 대한 사장님 직관**.

### v3 사업계획서가 가정한 것 (6장 BM)
- **수익 3축**: 구독 + 도구 유료화 + 매칭
- **구독 ARPU**: 가족 단위 월 9,900~19,900원 (retention × 유지개월)

### 사장님 신호가 함의하는 것
- 잇다 사용 패턴이 task-oriented(케어링 상황 때만 진입) → **월 단위 정기 결제와 정합성 약함**.
- 쏘카 비유와도 일관 — 차 쓸 때만 결제, 월 구독 아님.
- 그렇다면 BM 무게가 **도구 유료화 + 매칭으로 이동**해야 할 수 있음 (사용 시점 결제·건당 결제·완성물 결제).

### 별도 라운드 의제 (다음 사장님 결정 가용 시)
1. 구독 축을 v3 BM에서 **약화**할지 / 제거할지 / 유보하고 다른 축과 병행할지.
2. 도구 유료화의 구체 모델: 회고록 출판(건당)·체크리스트 프리미엄(건당)·AI 정리(횟수)·유언장 빌더 완성물(건당).
3. 매칭 의존도: v3 가설 d(매칭 파트너 계약)가 미검증인 채 구독 축이 약화되면 BM이 얇아짐 — 가설 d 검증 우선순위 올라감.
4. 가설 a(형제 케어링자 지불 의향)의 재해석: 구독 의향이 아니라 **task 시점 지불 의향**(케어 일지 기능에 한 번 지불 / 처방전 분석 횟수 결제 등).

### 별도 라운드 진행 방식 제안
- 전략 에이전트에 v3 6장(BM) 재검토 위임. 결과물: `docs/strategy/bm-revision-2026-06-XX.md`.
- 사장님 우선순위에 따라 일정 결정. 베타 테스트(D3·D4) 가동 중에 병행 가능.
