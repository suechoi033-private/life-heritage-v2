1. 레몬테라스 = 같은 카페 같은 사람이 "유언"과 "부모 케어" 둘 다 검색 → 한 입구가 자연
2. 운영비·복잡도 1배 (3인 팀에 (c)는 과부하)
3. 케어링 → 유언 자연 전환 (부모 케어하다 자기 준비로 흐름)
4. 헌장 일관 — "잘 죽는 것은 잘 사는 것" 단일 미션 유지

## 2. 두 얼굴의 정의

### 얼굴 1: 유언/웰니스 path
- **타깃**: 딩크·독신·싱글맘·미혼여성 + 본인 유언 의향 있는 모든 사용자
- **진입 카피**: *"내일 다시 못 깨어난다면, 가장 후회할 일은?"* (06-14 D 결정 카피)
- **핵심 동선**: 자기성찰 시리즈 → 일/사람 분기 → 유언 빌더(`note/will-builder.html`)·웰니스 기록
- **차별점**: **가족 협업 유언** (누구와 함께 작성·열람·이어받기) — 망고하다 1인 유언 차별

### 얼굴 2: 케어링 path
- **타깃**: 부모·가족 케어 중인 30~60대 (워킹맘·학부모·시니어 케어)
- **진입 카피**: *"오늘, [부모님 이름]에게 안부 한 줄 어떠세요"* (06-14 약속 카드 카피 계승)
- **핵심 동선**: 케어 대시보드(`care-dashboard.html`) + 케어 일지 + 긴급 카드(`care-emergency.html`)
- **차별점**: **가족 협업 기록** — 형제·자녀가 같이 보고 같이 쓰는 케어 기록

### 두 얼굴의 공통 부모
- 잇다 본채: 라이프 시리즈 · 마이 · 콘텐츠(forest.html) · 자기성찰
- 자연 전환: 케어링 path 활성 사용자가 라이프 시리즈에서 자기 유언으로 흐름

## 3. 진입 카드 — `index.html` 비회원 첫 화면

### 비회원 진입 (랜딩)

```
[히어로: 잇다 한 줄]
"잘 죽는 것은 잘 사는 것과 같다"

[진입 카드 1 — 유언/웰니스]
"내일 다시 못 깨어난다면, 가장 후회할 일은?"
→ 클릭 → ask/reflection 진입 (시리즈 1번)

[진입 카드 2 — 케어링]
"부모님께 오늘, 안부 한 줄"
→ 클릭 → 케어 페르소나 가입 (이름 입력 → care-dashboard)

[하단 작게]
"잇다 소개 보기" → 회사 소개/헌장 노출
```

**카드 위계 — 두 카드 동등 크기·색**. 사용자가 자기에게 맞는 카드를 고르게. 시스템이 권하지 않음(헌장 일관, 06-14 D6).

### 가입 후 (회원) 진입

- **답한 path 우선 노출** — 케어 답한 사람은 케어 대시보드 카드 우선, 유언 답한 사람은 약속/시리즈 우선
- **다른 path는 작게** — 마이·라이프에서 다른 path 활성화 가능
- **두 path 다 활성** — 두 카드 다 노출, 사용자가 선택

## 4. 가입 흐름

### 페르소나 선택 — 명시적 X
가입 화면(`signup.html`)에서 페르소나 선택하지 않음. **진입 카드가 곧 답한 path를 결정**.

- 카드 1(유언) 누르고 가입 → 유언 path 활성
- 카드 2(케어링) 누르고 가입 → 케어 path 활성
- 두 카드 다 답하면 두 path 다 활성

### 가입 데이터
- `profiles.entry_path` (신규 컬럼, nullable, 'will' | 'care' | null) — 첫 진입 카드 기록만
- `profiles.notification_pref` 그대로 활용
- 신규 테이블 0, 신규 컬럼 1개

| 면 | 망고하다 | 잇다 (b) 한 코드 두 얼굴 |
| :-- | :-- | :-- |
| 무게 | 변호사·법적 효력 강조 | 마음 정리부터 (법적 효력은 옵션) |
| 단위 | 1인 유언 중심 | **가족 협업 유언** (협업이 코어) |
| 진입 | 주민번호·간편비번 강제 (엔딩노트 사례) | 이메일·소셜만 (주민번호·위치 0) |

| 위험 | 신호 | 완충 |

## 10. 검증 — 3개월 후 (b)/(c) 재판단

다음 시점에서 데이터 보고 (b) 유지 vs (c) 분리 결정:

- **(b) 유지 신호**:
  - 두 path 다 활성 사용자 비율 ≥ 30%
  - 케어 → 유언 자연 전환율 ≥ 10%
  - 한 사용자 LTV가 단일 path 사용자의 1.5배 이상
  
- **(c) 분리 신호** (가능성 낮음):
  - 두 페르소나 완전 분리 (자연 전환 0%)
  - 한 path 사용자가 다른 path를 노이즈로 느낌 (정성 인터뷰)
  - 운영비 절감보다 단일 채널 acquisition 효율이 압도적

**기본 가정: (b) 유지**. (c)는 강한 반증 신호 있을 때만.
