# 결정 0002 — 지인 20명 검증을 어떻게 시작하고 기록할 것인가

- 일자: 2026-05-31
- 결정자: 창업자 / 전략 오케스트레이터
- 문 유형: **2-way door** (실행안·문서 범위. 시작 후 일정·페이스는 재조정 가능)
- 상태: 결정 완료, 실행 대기 (오늘 결정 5개 확정 후 D0 = 6/1 월 착수)

## 질문
사업계획서 §10의 "지인 20명 테스트"를 실제로 시작·완주하려면, 일정·기록·결정 게이트를 어디에 두고 어떻게 운영할 것인가?

## 배경
- 인터뷰 질문지(`interview-guide.md`)는 5/19에 완성됨 — 질문·판단기준은 이미 확정.
- 운영안 v2가 구글 독스로 5/27에 작성됨 — 골격(funnel·메시지·동선·KPI)은 잘 잡혀 있음.
- 그러나 실제 시작 0건. 구글 독스가 단일 원본이라 결정 게이트가 레포 작업 흐름과 분리되는 문제.
- 시드 콘텐츠 8편이 오늘 official 발행 완료 — 사용성 테스트 콘텐츠 즉시 사용 가능.

## 결정
1. **단일 원본은 `docs/interviews/EXECUTION-PLAN.md`** (본 결정문이 가리키는 문서). 구글 독스 v2는 히스토리 보존용으로만 둔다.
2. **D0 = 2026-06-01(월) 평일 시작.** 오늘(5/31 일)은 결정 5개 확정 + 구글 시트 세팅까지.
3. **PII 분리**: 명단·실명 = 구글 시트(창업자 소유) / 합산·인사이트·결정문 = 레포.
4. **검증 가능한 3축(케어링·콘텐츠·자기성찰)으로 사용성 테스트 한정**. 미완 영역은 "공사중 패스".
5. **결정 5개**(인원·시작·범위·20·40대·기록 위치)는 EXECUTION-PLAN.md §7에 권장안과 함께 정리. 창업자 ✓ 후 실행.
6. **주차 게이트**(W1~W4)를 건너뛰지 않는다. A 5 · B 5 · C 3 완료 전엔 "검증 완료"로 진행 못함.
7. **W4 종료 시점에 결정문 0003**으로 가설 a·b·c 확정/기각을 박제한다.

## 실행
- 본 결정문 + EXECUTION-PLAN.md 커밋·푸시.
- `CHARTER.md` §7 현재 단계 갱신.
- 창업자가 결정 5개 확정 → D0 명단 작성 시작.

## 트리거 — 이때 재검토
- W1 D4 KPI 미달(발송 <30, 완료 0) → 운영안 페이스 재설계
- A 또는 B 후보 5명 확보 실패 → "지인의 지인" 트랙 신설 또는 풀 외부 확장 검토
- W4 종료 후 판단기준 미달 → 페르소나 가설 자체 재검토(피벗 후보)
