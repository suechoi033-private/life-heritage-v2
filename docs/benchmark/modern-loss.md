# Modern Loss (modernloss.com) 벤치마킹

> 조사일: 2026-05-19
> 한 줄 요약: **콘텐츠 + 책 + 강연으로 라이프스타일 비즈니스에 안착한 미국 grief 미디어 brand. Product/tool 영역은 약함 = 잇다의 기회.**

## 1. 개요

| 항목 | 내용 |
|---|---|

| 창립자 | Rebecca Soffer (CEO) / Gabrielle Birkner (Executive Editor) |

### 창립 배경
- Soffer: 30세에 어머니 교통사고 → 4년 뒤 아버지 심장마비 (only child로 양친 사망)
- Birkner: 아버지·새어머니 가택침입 강도 살해
- 둘이 "Women With Dead Parents" 비공식 디너에서 만남
- 둘 다 첫 아이 임신 말기에 사이트 런칭, TheLi.st 무료 사무공간 사용

### NYT가 만든 카피
2014년 3월 "An Online Generation Redefines Mourning" 기사 이후 **"redefining mourning"**이 모든 PR의 핵심 카피.

## 2. 콘텐츠 전략

### 구조 (WordPress)
- My Loss (Personal Essays) — 코어
- Features (인터뷰·Q&A)
- Advice (가이드)
- Resources (디렉토리·법무·치료)
- Blog
- Our Book / Handbook
- Share Your Story → Submittable 연동

### 형식 규칙 (Submittable guideline)
- **원본 700~800단어** (한국어 약 1,400~1,800자)
- personal essay / listicle / Q&A / opinion / reported piece
- 오디오·카툰·일러스트 환영
- 시·픽션 받지 않음
- "한 grief 측면에 좁게 초점" — 회고록 전체 여정 NO

### 톤
- "Bit of attitude; zero platitude" — 종교색·자조서 톤 의식적 거부
- 유머·자조·블랙코미디 허용
- Stephen Colbert 추천사: "not your mother's death and dying book"
- 거부: "신이 또 다른 천사를 데려갔다", "6개월 지났으니 그만 슬퍼하라" 같은 클리셰

### 대표 시리즈
- Sheryl Sandberg 인터뷰 ("Don't Call Sheryl Sandberg a Widow")
- "Light After Loss" — 1-800-Flowers 후원 Facebook Live
- "My Grief is F*cking Funny"
- "trigger day" 콘텐츠 — 첫 명절, deathiversary, diagnosisversary
- "Workplace Grief" 시리즈

### 발행 빈도
- 전성기 본 사이트 주 2~3편 (추정)
- 2022 이후 Substack으로 무게중심 이동, 본 사이트 둔화

## 3. 콘텐츠 생산 모델

- **잡지형 pitch-edit** (자유 게시판 아님)
- 외부 contributor pitch → Submittable 폼 → 에디터 검토 → 게재
- 강한 큐레이션, 분량 제한, 주제 제한
- 명사 contributor pipeline: Lucy Kalanithi, Amanda Palmer, Brian Stelter, Cynthia Nixon, Jessica Williams, Rachel Bloom 등
- 코어 staff 5인 미만 추정
- 사용자 직접 글 작성 기능 없음 (Substack 유료 구독자만 thread 시작 가능)

- **Modern Loss (2018, Harper Wave)**: 40+ contributor anthology
- **Modern Loss Handbook (2022, Running Press)**: 인터랙티브 워크북 — Gayle King favorite

## 8. 한계 / 약점

1. **콘텐츠 → 도구 전환 미완성** — Handbook이 종이책에 머무름
2. **transactional/lifecycle 도구 부재** — 유언, 디지털 유산, 추모 사이트 같은 "행위" 기능 없음
3. **B2B 약함** — 잇다의 white space
4. **본 사이트 → Substack 이동의 위험** — IP가 외부 플랫폼에 lock-in되어 가는 구조
5. **lasting community 약함** — Substack thread만 있음

## 9. 잇다에 적용할 lessons

### 베껴야 할 것
1. **"Bit of attitude; zero platitude" 톤** — "위로 말고, 같이" 같은 한국식 카피
2. **창립자 personal narrative** — 모든 마케팅의 backbone
3. **명사 contributor pipeline** — 런칭 6개월간 매주 한 명씩 publish
4. **trigger day 시리즈** — 한국식: 49재·100일·1주기·명절 제사
5. **잡지형 pitch-edit** — Submittable식 + 분량 제한 + 강한 큐레이션
6. **컨퍼런스·강연 backbone** — 삼성·우아한형제들·당근 사내 워크숍, 국립중앙도서관
7. **책으로 가는 path** — (a) 한국인 50명 죽음 에세이 anthology → (b) "내 죽음을 미리 준비하는 워크북"

### 피해야 할 것
1. **수익 모델 후행 설계** — Soffer 본인 자백. 잇다는 런칭 시점부터 매출 3축 명확
2. **콘텐츠 only stack** — 콘텐츠는 acquisition channel, product 본체 아님
3. **본 사이트 ↔ 외부 플랫폼 이원화** — 1st party data·SEO를 자체 채널에 lock
4. **plateau에 안착** — 라이프스타일 비즈니스 함정

### 잇다가 더 잘할 수 있는 것 (Modern Loss white space)
1. **사전 준비(advance) 카테고리 owner** — Modern Loss는 사후만
2. **interactive tool / 앱** — Handbook 콘텐츠를 앱·체크리스트·vault로
3. **장례·보험·법무 B2B 파트너십** — 미국 거의 없음
4. **한국 의례 캘린더** — 명절·제사·49재·100일 native
5. **lasting community layer** — 앱 내 anonymous peer circle
6. **공공·의료 파트너십** — 호스피스·상조·종교계·서울시
7. **한겨레/중앙 시그니처 feature** — 창립 6개월 내 PR 1순위 KPI
