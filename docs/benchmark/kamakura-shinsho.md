# 鎌倉新書(카마쿠라신쇼, Kamakura Shinsho) 벤치마킹

> 조사일: 2026-05-21
> 한 줄 요약: **倒産(도-산 = 도산) 직전의 불교 전문 출판사가 "정보 가공업"으로 自己(자기) 재정의 → 終活(슈카츠 = 임종 준비) 포털 매칭 사업으로 전환 → 東証プライム(도-쇼 프라임 = 도쿄증권거래소 프라임) 상장. 終活ねっと(슈카츠넷)가 망한 단일 매칭 모델과 정반대로, 出版(출판)·매칭·B2B·官民協働(관민협동)으로 다리를 여러 개 깐 "잇다가 가장 닮아야 할 생존자".**

> ※ 終活(슈카츠) = 인생 마지막을 준비하는 활동. 한국어로는 "임종 준비" 또는 "사전 정리".
> ※ 供養(쿠요-) = 죽은 이를 기리는 행위(장례·묘·불단)를 묶는 일본식 개념. 카마쿠라신쇼의 본업 3사업(葬祭·お墓·仏壇)을 "供養 3사업"이라 부름.
> ※ 官民協働(칸민쿄-도- = 관민협동) = 행정(자치단체)과 민간기업이 함께 주민 서비스를 제공하는 사업. 카마쿠라신쇼의 최신 성장 엔진.
> ※ 환율 환산은 종전 문서와 동일하게 1엔 ≈ 10원으로 단순 환산(개략치).

## 2. 사업 구조 (供養 3사업 + 확장 4축)

### 供養(쿠요- = 공양) 3사업 — 본업

| 사업 | 서비스 | 내용 |
|---|---|---|
| **お墓(오하카 = 묘) 사업** | **いいお墓(이이 오하카)** | 全国(젠코쿠 = 전국) 霊園·墓地(레이엔·보치 = 공원묘지·묘지) **약 1만 건+ 게재** (e-ohaka.com). 매출 비중 최대(약 38%) |
| **葬祭(소-사이 = 장례) 사업** | **いい葬儀(이이 소-기)** | 葬儀(장례) 상담·依頼(이라이 = 의뢰) 매칭. **누적 상담 100만 건+ 돌파.** 提携(제휴) 葬儀社(소-기샤 = 장례 회사) 약 1,000사, 提携 斎場(사이조- = 장례식장) 약 6,500곳 (2024년 기준) |
| **仏壇(부츠단 = 불단) 사업** | **いい仏壇(이이 부츠단)** | 仏壇店(부츠단텐 = 불단 가게) 매칭. 가장 오래된 포털 중 하나 |

### 확장 4축 — 슈카츠넷이 못 깐 다리들

| 축 | 내용 |
|---|---|
| **相続(소-조쿠 = 상속) 사업** | **いい相続(이이 소-조쿠)** — 税理士(제이리시 = 세무사)·士業(시교- = 전문 자격사) 매칭. YMYL이지만 전문가 連携(렌케이 = 연계)로 처리 |
| **介護(카이고 = 개호/노인 돌봄) 사업** | 介護(개호) 시설·서비스 정보 매칭 |
| **官民協働(관민협동) 사업** | 自治体(지자체)와 終活(슈카츠) 연계 — 후술(섹션 4). 최신 고성장 엔진 |
| **B2B / 出版(출판) / 보험** | 葬儀社(장례 회사)용 B2B 도구·業界誌(교-카이시 = 업계지) 출판·終活(슈카츠) 보험 등 |

## 3. 비즈니스 모델

### 핵심 — "供養·終活판 価格.com(카카쿠코무 = 가격비교 사이트)"

1. **無料掲載(무료 게재) + 成約手数料(세이야쿠 테스-료- = 성약 수수료)**
   - 葬儀社(장례 회사)·石材店(세키자이텐 = 석재점)·仏壇店(불단 가게)·寺院(지인 = 사찰) 정보를 **무료로 게재**
   - 高齢者(코-레이샤 = 고령자)와 그 가족을 紹介(쇼-카이 = 소개) → **成約(성약) 시 수수료** 수취
   - = 価格.com / リクルート(리크루트)식 매칭·送客(소-캬쿠 = 고객 송출) 모델
2. **官民協働(관민협동) 수익** — 自治体(지자체) 대상 冊子(삿시 = 책자)·시스템·운영 지원 (B2G)
3. **出版·B2B·보험** — 業界誌(업계지), 葬儀社(장례 회사)용 도구, 終活(슈카츠) 보험

### 핵심 차별점 (vs 슈카츠넷)

- **40년 業界(교-카이 = 업계) 신뢰** — 출판사 시절부터 葬儀(장례)·仏具(불구) 업계와의 관계 자산
- **多角化(타카쿠카 = 다각화)** — 매칭 단가가 코로나·家族葬(카조쿠소 = 가족장) 추세로 떨어져도 다른 다리가 받쳐줌
- **B2G(官民) 진입 장벽** — 自治体(지자체) 계약은 한번 깔리면 경쟁자가 못 들어옴(switching cost ↑)

## 4. 官民協働(칸민쿄-도- = 관민협동) 사업 — 최강 海(해)자

> 잇다가 가장 주목해야 할 부분. 슈카츠넷에 없던, 베끼기 어려운 護城河(고-조-가 = 해자/moat).

| 항목 | 내용 |
|---|---|
| 개시 | **2021년 3월** |
| 규모 | 提携(제휴) **自治体(지자체) 500곳+ 돌파** (전국 인구의 **약 60% 커버**) |
| 성장률 | FY2025 매출 5억 엔(약 50억 원) 수준, **+58.3% 고성장** |
| 배경 | 40년 終活(슈카츠) 知見(치켄 = 지견/노하우)을 행정에 제공 |

### 주요 메뉴

1. **おくやみコーナー(오쿠야미 코-나- = 사망 후 행정 수속 원스톱 창구) 개설·운영 지원**
   - 死亡後(시보-고 = 사망 후) 흩어진 행정 수속을 한 창구에서 안내 — 시스템 제공 + 設置(셋치 = 설치) 지원 + 運用効果測定(운용 효과 측정)
2. **終活(슈카츠) 冊子(책자) 무상 제작·제공**
   - おくやみハンドブック(오쿠야미 핸드북 = 사망 안내 핸드북), エンディングノート(엔딩 노트), 終活べんり帳(슈카츠 벤리초- = 종활 편리장), 空き家ガイドブック(아키야 가이드북 = 빈집 가이드), 相続(상속)·介護保険(개호보험) 가이드 등
3. **終活連携協定(슈카츠 렌케이 쿄-테이 = 종활 연계 협정)**
   - 終活(슈카츠) 세미나·**終活相談ダイヤル(슈카츠 소-단 다이야루 = 종활 상담 다이얼)** 제공
   - "엔딩 노트만 쓰게 하고 끝내지 않는" 토털 서포트 — 職員(쇼쿠인 = 공무원) 부담 경감 + 住民(주민) 안심

→ **핵심 통찰**: 행정이 終活(슈카츠) 啓発(케이하츠 = 계몽)·遺族(이조쿠 = 유족) 지원에 골머리를 앓는다 → 카마쿠라신쇼가 컨텐츠+시스템을 통째로 대행 → 住民(주민)이 자연스럽게 自社(자사) 포털 매칭으로 유입. **B2G가 B2C 퍼널의 入口(이리구치 = 입구)가 되는 구조.**

## 7. 잇다에 적용할 lessons

### 베껴야 할 것

1. **"정보 가공업" 재정의 = 잇다의 정체성 모델.** 카마쿠라신쇼는 "출판사"라는 형식에 갇히지 않고 "가치 있는 정보를 잇는 업"으로 재정의해 살아남음. 잇다도 "앱"이 아니라 "사람·정보·결정을 잇는 lifecycle 업"으로 정의.
2. **다리를 처음부터 여러 개**: 매칭 단일 = 死(사). 카마쿠라신쇼처럼 **매칭 + 회원/도구 + B2B + B2G(官民)** 4축. (종전 슈카츠넷 교훈과 一致 = 일치.)
3. **官民協働(관민협동) = 한국 버전 검토 가치 大.** 한국 지자체도 고독사·무연고·遺品整理(이힌세이리 = 유품 정리)·사망 수속에 골머리. **おくやみコーナー(사망 후 원스톱 창구)**·終活(슈카츠) 책자·상담 다이얼 모델은 한국 행정에 그대로 이식 가능 → B2G가 B2C 퍼널 入口.
4. **"いい○○(좋은 ○○)" 카테고리 지배 + 무료 게재 + 성약 수수료** 매칭 구조는 검증된 흑자 모델.
5. **조사 PR 자산화** — "장례 비용 실태" 같은 자체 통계 = 매년 SEO 권위 + 언론 인용. (잇다도 "한국 임종 준비 실태" 정례 조사 보유 검토.)

### 한국 vs 일본 차이

6. **行政(행정) 데이터 연계 난이도**: 한국 지자체 사망 신고·상속 수속은 정부24·홈택스 등에 분산. 일본 おくやみコーナー처럼 통합 대행하려면 행정 협업·개인정보 처리 설계 필요.
7. **검색 다채널화**: 일본은 Google SEO 1축으로 충분했으나, 한국은 Naver(네이버)+Kakao(카카오)+YouTube 동시 필수(슈카츠넷 문서 교훈과 동일).

### 잇다가 카마쿠라신쇼보다 잘할 수 있는 것

8. **App-native lifecycle**: 카마쿠라신쇼는 web 포털·매칭 중심. 잇다는 푸시·매년 업데이트·**가족 공유**로 "결정 → 기록 → 가족 전달"까지 묶을 수 있음.
9. **돌봄(care) 통합**: 카마쿠라신쇼의 介護(개호)는 매칭에 그침. 잇다는 4050 자녀 돌봄형 타깃에 care-dashboard·emergency를 native 통합(현 잇다 제품 강점).
10. **AI 활용**: 1차 콘텐츠 초안 + 전문가 監修(칸슈 = 감수) sign-off 워크플로우. (YMYL이므로 슈카츠넷 교훈대로 감수 必須.)
