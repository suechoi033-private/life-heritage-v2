2. 발견된 문제

HIGH아이콘–라벨–행선지 3중 불일치 (📚)

책 아이콘 + title "안내"는 라이브러리/도움말을 暗示하지만 실제는 "죽음 준비" 카테고리 콘텐츠 피드. 기대와 현실이 어긋나고, 무거운 주제를 맥락 없이 노출.

HIGHdeath_prep 카테고리 고정 직행 (📚)

글로벌 상단 아이콘이 특정 카테고리로 직행. 비로그인/첫 방문 첫인상에서 부담. infoIcon 기본 노출이라 가치제안 전에 죽음준비로 보냄.

MED하단탭 커뮤니티와 중복 (📚)

도착 화면이 커뮤니티 탭과 사실상 동일(필터만 다름). 상단 아이콘만의 고유 가치 부재.

MED불필요한 경유 리다이렉트 (✏️)

seed.html을 한 번 로드 후 즉시 diary-write로. 깜빡임 + 뒤로가기 히스토리 꼬임. diary-write 직접 링크가 맞음.

MED맥락 무시 단일 기본값 (✏️)

전 화면에서 ✏️=일기 고정. 커뮤니티/콘텐츠 화면에선 포스트 작성을 기대. 단일 기본값이 좁음.

HIGH상단 우측 자리의 정체성 부재 (IA)

한쪽은 카테고리 딥링크, 한쪽은 단일 작성 액션 — 묶는 멘탈모델 없음. 보통 이 자리는 검색/알림/저장/작성 중 하나의 일관된 역할.

HIGH작성 진입점 분산 (IA)

홈 ✏️(일기) · 라이프 FAB(일기/버킷) · 커뮤니티 FAB(콘텐츠/포스트 시트) · 케어링 기록 버튼 — 화면마다 "쓰기"가 다른 의미. 멘탈모델 흩어짐.

3. 제안 옵션 (결정 전)

📚 자리의 재정의 (택1)

A1 · 진짜 "가이드"
  • 서비스 사용법 + 입문 큐레이션
  • 라이프/케어링/죽음준비 시작하기
  • "안내" 라벨과 일치
A2 · "내 서재/저장" ★
  • 북마크·내 글·내 기록 모아보기
  • 책 아이콘과 가장 자연스러움
  • 재방문 가치↑
A3 · 제거
  • 커뮤니티 탭과 중복이면 정리
  • 상단을 검색으로 대체 검토

✏️ 자리의 재정의 (택1)

B1 · 직접 연결
  • diary-write로 바로(리다이렉트 제거)
  • 최소 수정·즉시 가능
B2 · 통합 작성 시트 ★
  • 탭 → "무엇을 쓸까요?"
  • 일기 · 버킷 · 커뮤니티 글
  • 전 화면 공통, 분산 수렴
B3 · 맥락형
  • 화면별 기본 타입
  • 홈=일기, 커뮤니티=포스트

4. 전략 세션용 열린 질문

Q1. 상단 우측은 무엇을 위한 자리인가? (검색 / 알림 / 저장(서재) / 작성 / 도움말 — 하나로 정해야)
Q2. 📚의 의도된 목적은? death_prep 고정이 의도였나, 임시였나?
Q3. 작성(쓰기) IA를 하나로 통일할 것인가, 화면별 특화할 것인가?
Q4. 비로그인 첫인상에서 무엇을 강조할 것인가? (지금은 death_prep로 보내 부담)
Q5. 전역 검색은 어디에? (콘텐츠가 쌓일수록 진입점 필요 — 현재 없음)
